# 1장 빅데이터 개요
## 1.1 빅데이터의 등장 배경
### 1.1.1 데이터 환경의 변화
* 인터넷, SNS, 스마트폰 보급 등으로 정형·비정형 데이터 폭증
* ZB(제타바이트) 시대 진입: 2020년 기준 2011년 대비 50배 증가
### 1.1.2 ICT 기술 발전
* 메인프레임 → PC → 모바일 → 스마트 시대 도래
* 증강현실, 사물정보통신, 인공지능 등 기술 발전과 데이터 증가
## 1.2 빅데이터의 정의
### 1.2.1 일반적 정의
* 기존 수단으로 수집, 저장, 분석, 시각화가 어려운 대규모 정형/비정형 데이터
### 1.2.2 기술적 정의
* 수집부터 분석까지 포함한 정보통신기술
* 가치 추출 및 분석을 위한 차세대 기술·아키텍처
## 1.3 빅데이터의 유형
### 1.3.1 정형 데이터
* 관계형 DB, 스프레드시트, CSV 등 구조 고정
* 용이한 검색, 갱신, 삭제 등 연산 가능
### 1.3.2 반정형 데이터
* HTML, XML, RDF, JSON 등 구조 정보 포함
* 스키마 수정 용이, 파일 단위 구성
* 정형 데이터로 변환 가능하나 질의 처리 어려움
### 1.3.3 비정형 데이터
* 구조 없음: 텍스트, 오디오, 이미지, 영상 등
* NLP, 전처리 필요, 머신러닝 입력 데이터로 활용
* 사회적·문화적 맥락 반영, 마케팅/BI에 중요
# 2장 빅데이터 속성
## 2.1 공통적 특성 3V
### 2.1.1 Volume(규모)
* TB\~ZB 이상의 방대한 양
### 2.1.2 Velocity(속도)
* 실시간 수집, 저장, 분석, 시각화 중요
### 2.1.3 Variety(다양성)
* 정형, 반정형, 비정형 데이터 모두 포함
## 2.2 확장 특성
### 2.2.1 Veracity(진실성)
* 수집 시 오류 제거, 전처리 통한 신뢰성 확보
### 2.2.2 Value(가치)
* 분석 결과가 의사결정에 기여할 수 있어야 함
### 2.2.3 Variability(가변성)
* 맥락에 따른 데이터 의미 변화 고려 필요
### 2.2.4 Visualization(시각화)
* 분석 결과를 이해하기 쉬운 형태로 제공
# 3장 빅데이터 처리과정
## 3.1 생성과 수집
* 내부 데이터, 외부 데이터, 센서 데이터 등
* 자동/수동 수집 방식, 개인정보 주의 필요
## 3.2 저장
* 데이터 유형에 따른 저장 방식 필요
* 분산파일시스템(HDFS), NoSQL, 병렬 DBMS, 네트워크 저장 시스템
## 3.3 처리
* 일괄처리, 실시간 처리, 전처리 및 정제 과정 포함
* 오류 제거, 분석 목적의 재배치 수행
## 3.4 분석
* 통계, AI 기법 활용: 데이터 마이닝, 군집화, 분류 등
* SQL 기반 정형 분석도 포함
## 3.5 표현 및 활용
* 시각화 도구 사용(R, Python, Tableau)
* 외부 제공 시 프라이버시 고려
## 3.6 폐기
* 사용 기한 이후 안전하게 논리/물리적 삭제
* 오버라이팅, 디가우싱 등 활용
# 4장 빅데이터 처리과정별 기술
## 4.1 수집 기술
* 로그수집기, 센싱, 크롤링, RSS 리더, 오픈 API
* ETL: 다양한 소스 → 공통 포맷 변환 및 적재
## 4.2 저장 기술
### 4.2.1 Hadoop
* HDFS + MapReduce
* 낮은 비용, 빠른 처리 속도
### 4.2.2 NoSQL
* 비정형 데이터 저장에 유리, 확장성 우수
* MongoDB, HBase, Cassandra 등
## 4.3 처리 기술
### 4.3.1 일괄 처리
* Hadoop MapReduce, Google MapReduce
### 4.3.2 실시간 처리
* IBM InfoSphere Streams, Twitter Storm
### 4.3.3 프로그래밍 언어 지원
* Google Sawzall, Hadoop Pig 등
## 4.4 분석 기술
### 4.4.1 텍스트마이닝
* 비정형 텍스트 분석, 연계성 파악
### 4.4.2 웹마이닝
* 웹에서 수집한 데이터 분석
### 4.4.3 기계학습
* 학습을 통한 예측/분류
### 4.4.4 감성분석
* 문장 의미 해석 및 긍정/부정 분류
# 5장 빅데이터 보안 개념
## 5.1 개인정보 보호 이슈
* 수집\~폐기 전 과정에서 개인정보 유출 우려
* 데이터 생성 주체의 동의 필요
## 5.2 처리과정별 보안
### 5.2.1 수집 단계
* 접근 통제, 웹 크롤러 제한 설정 필요
### 5.2.2 저장 단계
* 암호화 저장, 접근 통제, 등급 분류 필요
### 5.2.3 처리·분석 단계
* 익명화, 암호화된 데이터의 분석 수행
* 수집 목적 외 사용 금지
### 5.2.4 표현 단계
* 시각화 정보의 유출 위험 고려
* 제3자 무단 제공 금지
### 5.2.5 폐기 단계
* 사용 후 데이터 완전 삭제 필요
* 디가우싱, 천공 등 물리적 폐기 고려
