# 1장 빅데이터 수집 단계의 보안 기술
## 1.1 수집 단계 보안 위협
### 1.1.1 무분별한 데이터 수집
- 명시적 동의(Opt-In)와 관계없이 무분별하게 수집되는 경우 존재
	- Opt-In: 데이터 수집자가 정보 주체에게 수집 동의를 받아야 수집이 가능한 방식
		- 국내의 경우, 모든 데이터에 Opt-In 의무화
	- Opt-Out: 사용자가 동의 의사를 표현하지 않더라도 수집에 동의했다고 가정하는 방식
- 문제점
	- 악의적인 수집가: 고품질의 데이터를 위해 정보 주체의 정보 주권 침해
	- 정보 주체: 다양한 서비스를 이용하면서 데이터 수집 당하는 것을 인지 못함
### 1.1.2 데이터 유출
- 명시적 동의를 통해 수집된 정보라도 데이터 유출 위험 존재
### 1.1.3 데이터 중요성 인식 부족
- 개인정보에 대한 낮은 인식 수준
## 1.2 수집 단계 보안 위협 대응 기술
### 1.2.1 개인정보보호 데이터 수집(PPDC)
- Privacy-Preserving Data Collection(PPDC)
- 데이터 수집 단계에서 개인정보보호를 달성하기 위한 모든 기술들
- 데이터 교란 기법
	- 데이터를 교란하여 **고품질의 데이터를 얻지 못하게 방해**하는 것
- 동형 암호(Homomorphic Encryption) 기법
	- 암호화된 데이터로 연산을 해도 복호화한 데이터에서 연산을 한 것과 동일한 결과가 나오는 암호 기술
### 1.2.2 정보보호 교육 및 캠페인
- 정보 주체의 데이터 중요성 인식 부족 위협과 데이터 유출에 대응할 수 있는 보안 기술
### 1.2.3 관련 법과 제도 마련
### 1.2.4 통신 보안
- 데이터를 안전하게 송수신할 수 있는 기술
- VPN
- SSL/TLS 기술
# 2장 빅데이터 저장 단계의 보안 기술
- 데이터 관리자: 데이터 수집가보다 데이터에 대해서 높은 보안 수준을 유지해야할 의무
## 2.1 저장 단계 보안 위협
### 2.1.1 데이터 무결성 훼손
- 주로 제 3의 공격자나 내부자에 의해 발생
- 데이터 품질 저하 및 분석 단계에 악영향
### 2.1.2 데이터 유출
- 빅데이터 lifecycle의 모든 곳에서 발생 가능
- **내부자에 의한 의도적인 데이터 유출** 가능 -> 탐지가 어려우므로 **적극적인 관리/감독** 필요
### 2.1.3 시스템 가용성 위협
- 올바른 사용자가 시스템을 사용하고 싶을 때 사용할 수 있도록 가용성을 보장하는 것이 중요
## 2.2 저장 단계 보안 위협 대응 기술
### 2.2.1 접근 제어
- 무결성 훼손, 유출 위협을 최소화할 수 있는 기술
- DAC, MAC, RBAC, ABAC 기반 권한 부여
### 2.2.2 시스템 분산화
- 시스템 가용성을 위한 기술
- 빅데이터는 양이 방대하므로 **분산화된 클라우드**를 통해 관리
	- 클라우드: 고가용성 제공
### 2.2.3 보안 감사
- 사용자가 수행한 작업들을 로그에 저장하는 방법인 보안감사를 토대로 모니터링
- 주기적인 보안 감사 수행 필요
### 2.2.4 암호화
- 유출되더라도 기밀성 보호를 위한 것
- 공개키, 대칭키(속도가 빨라 대용량 데이터 암호화에 적합)
- 속성기반 암호화: 암호화 + 접근제어, 적합한 속성을 가진 사용자만 데이터를 암호화 하도록 하는 기술
	- 접근제어 정책(속성 트리)의 존재 위치에 따라 분류
	- Ciphertext-Policy(CP)-ABE
	- Key-Policy(KP)-ABE
- 대칭키 기반 암호화, 비대칭키 기반 CP-ABE, KP-ABE 활용
# 3장 빅데이터 분석 단계의 보안 기술
## 3.1 분석 단계 보안 위협
### 3.1.1 개인 식별 가능성
- 개인 식별
	- 특정 개인을 개인 식별 정보를 통해 식별
	- 재식별: 개인 식별 정보가 제거되었더라도 다른 데이터와 결합해서 다시 특정 개인을 식별
- 개인 식별 정보(Personal Identifiable Information, PII)
	- 정보 주체를 특정하여 식별할 수 있는 모든 정보
- 분석 단계에서 가장 중요하고 무조건 최소화되어야하는 위협
- 개인정보 처리자와 데이터 분석가 모두를 통해 개인 식별될 수 있음
	- 개인정보 처리자: 개인정보를 처리하기 전 원본 데이터를 통해 개인 식별 가능
	- 데이터 분석가: 다양한 데이터 마이닝 기법을 통해 의도적/의도치 않은 개인 식별 가능
### 3.1.2 데이터 유출
## 3.2 분석 단계 보안 위협 대응 기술
### 3.2.1 비식별화(De-Identification)
- **개인 식별 정보(PII)**를 총계처리, 데이터 마스킹, 가명화 등을 통해 제거
- **데이터 집합에서 개인 식별 정보를 제거**함으로써 개인을 식별할 수 없게 하는 기술
- 가명처리
- 총계처리
- 데이터 삭제
=> 의도적으로 비식별화를 하지 않을 가능성 존재
=> K-익명성, I-다양성을 통해 비식별화의 적절성 평가
### 3.2.2 프라이버시 보존형 데이터 마이닝(PPDM)
- Privacy Preserving Data Mining
- 데이터 분석 과정에서 **특정 개인을 식별할 수 없도록 하는 기술**
- 데이터 교란을 발생시키는 방법
- 프라이버시와 데이터 효용성 사이에서 균형을 적절히 맞추는 게 중요
### 3.2.3 보안 감사
- 데이터 분석가의 **로그를 남겨 주기적 행적 조회 및 악의적 행동 탐지**
# 4장 빅데이터 활용 단계의 보안 기술
## 4.1 활용 단계 보안 위협
### 4.1.1 재식별
- 분석 단계보다 재식별에 의한 개인 식별이 더 빈번하게 발생
## 4.2 활용 단계 보안 위협 대응 기술
### 4.2.1 프라이버시 보호 모델
- 비식별화된 데이터를 평가하기 위한 평가 기준
	- 1단계(사전검토) -> 2단계(비식별 조치) -> 3단계(적정성 평가) -> 4단계(사후 관리)
	- 식별자: **PII**, ex) 이름, 주소, 성별
	- 준식별자: **다른 정보와 결합해서 충분히 개인을 식별할 가능성이 있는 데이터**
- K-익명성
	- **Linking Attack**으로부터 재식별을 막기 위해 등장
	- 주어진 데이터 집합에서 동일한 값이 최소 K개 이상 존재하도록 해서 개인을 익명화
	- 취약점
		- 배경지식에 의한 공격으로 뚫릴 가능성
- I-다양성
	- 데이터 집합에서 함께 익명화되는 레코드들은 **적어도 I개 이상 서로 다른 데이터를 가지도록 하는 것**
	- 취약점
		- 쏠림 공격
		- 유사성 공격
- T-근접성
	- 데이터 집합에서 **특정 데이터 분포가 전체 데이터 집합에서의 분포와 비교해 T 이하의 차이가 보이도록** 하는 것 => 고르게 분포되도록 하는 것
# 5장 빅데이터 폐기 단계의 보안 기술
## 5.1 폐기 단계 보안 위협
### 5.1.1 데이터 유출
- 데이터 잔존에 의한 위협
	- **폐기되어야 할 데이터가** 폐기되지 않고 **데이터 저장소에 그대로 유지**
## 5.2 폐기 단계 보안 위협 대응 기술
### 5.2.1 오버라이팅(Overwriting)
- 기존 데이터에 여러 번 덮어 써서 원래 데이터를 알지 못하게 하는 기술
- 소프트웨어적 특징 보유. 클라우드 분산화 속성 때문에 **오버라이팅을 수행해야 할 데이터 위치를 정확히 알기 어려움**
### 5.2.2 디가우징(Degaussing)
- 자기장을 이용한 물리적 파괴 => 디스크 재사용 불가
