# 1장 빅데이터 개요
## 1.1 빅데이터
### 1.1.1 등장 배경
* 인터넷, SNS, 스마트폰 보급 등으로 정형·비정형 데이터 폭증
### 1.1.2 ICT 기술 발전에 따른 데이터의 변화 방향
* ZB(제타바이트) 시대 진입: 2020년 기준 2011년 대비 50배 증가
- 정형 데이터 -> 비정형 데이터 -> 사물 정보, 인지 정보
- 구조화 -> 다양성, 복합성, 소셜 -> 현실성, 실시간성
### 1.1.3 정의
- 맥킨지 보고서
	- 기존 데이터에 비해 너무 커서 수집, 저장, 검색, 분석, 시각화 등이 어려운 정형, 비정형 데이터
- TTA 정보통신용어사전
	- 방대한 양의 정형, 반정형, 비정형 데이터의 집합을 수집, 저장, 관리, 분석, 시각화하는 정보통신 기술 분야
- 기술 분야 측면에서의 정의
	- 다양한 종류의 대규모 데이터로부터 가치를 추출하고, 데이터의 수집, 발굴, 분석을 지원하도록 고안된 차세대 기술
=> **대용량의 정형, 반정형, 비정형 데이터의 집합**
## 1.2 빅데이터의 유형
### 1.3.1 정형 데이터(Structured Data)
- **미리 정해 놓은 구조**에 따라 저장되도록 구성된 데이터
* 관계형 DB, 스프레드시트, CSV 등 구조 고정
* 용이한 검색, 갱신, 삭제 등 연산 가능
### 1.3.2 반정형 데이터(Semi-Structured Data)
- **구조가 변경**될 수 있는 데이터
* HTML, JSON 등 구조 정보 포함
- 파일 내부의 형식 정보를 통해 정형 데이터로 변환 가능
* 스키마 수정 쉬움, 파일 단위 구성되어 공유가 쉬움
* 질의 처리 어려움
### 1.3.3 비정형 데이터
* 구조 없음: 텍스트, 오디오, 이미지, 영상 등
* 데이터의 특징을 추출해 반정형/정형 데이터로 변환하는 전처리가 필요
	* 텍스트 - Natural Language Processing
	* 미디어 파일 - Machine Learning을 위한 입력 데이터로 활용
* 사회적·문화적 맥락 반영, 마케팅/BI에 중요
# 2장 빅데이터 속성
## 2.1 공통적 특성 3V
### 2.1.1 Volume(데이터의 크기)
- 단순 저장되는 물리적인 데이터 양
* TB 이상의 방대한 양
### 2.1.2 Velocity(속도)
* 실시간 수집, 저장, 분석, 시각화 중요
### 2.1.3 Variety(다양성)
* 정형, 반정형, 비정형처럼 다양한 데이터를 포함
## 2.2 확장 특성 4V/5V/6V/7V
### 2.2.1 Veracity(진실성)
- 가치 있는 결과를 만들기 위해 빅데이터는 신뢰할 수 있어야 함
### 2.2.2 Value(가치)
* 분석 결과가 의사결정에 활용될 수 있을만큼 가치가 있어야 함
### 2.2.3 Variability(가변성)
* 맥락에 따른 데이터 의미 변화 고려해서 원래의 의미가 반영되도록 노력해야 함
### 2.2.4 Visualization(시각화)
* 분석 결과를 이해하기 쉬운 형태로 제공

=> **빠른 속도**로 생성되는 **대량**의 **다양한 데이터**를 **정확성**을 유지하도록 가공하고, 맥락에 따른 **가변성**을 고려한분석을 통해 **가치**있는 결과를 도출해서 이를 이해하기 쉽게 **시각화**하여 제공
# 3장 빅데이터 처리과정
- Big Data Life Cycle
## 3.1 생성과 수집
- **데이터 소스**와 관련이 있는 빅데이터 생성
	- 데이터 소스
		- DBMS의 **내부데이터**
		- SNS나 공공데이터 같은 **외부데이터**
		- 다양한 디바이스/센서를 통해 데이터 수집
* 수집 단계
	* 내부 데이터 수집
		* DBMS나 센서 등에서 **정형 데이터** 수집
	* 외부 데이터 수집
		* 인터넷을 통해 **정형, 반정형, 비정형 데이터** 수집
### 3.1.1 대표적인 수집 방법
- 크롤링, 로그 수집기, 센싱, 오픈 API 등
- 개인정보 수집 시 주의점
	- 사전에 데이터 사용에 대한 허가 필요
## 3.2 저장
* 수집한 데이터의 형태에 따라서 빅데이터 시스템에 적절하게 저장하는 단계
* 분산파일시스템(HDFS), NoSQL, 병렬 DBMS, 네트워크 구성 저장 시스템
- 분산 파일 시스템, NoSQL, 병렬 DBMS, 네트워크 구성 저장 시스템
## 3.3 처리
- 수집한 데이터들을 실시간으로 일괄 처리하는 데이터 정제/전처리 과정
- 신뢰성 확보를 위한 오류 데이터 제거
- 맵 리듀스, 하둡, R
## 3.4 분석
- 데이터 안의 유용한 정보를 찾아내기 위해 데이터를 가공
- 데이터 마이닝, 머신러닝, 분류, 군집화, 감정 분석
## 3.5 표현(시각화) 및 활용
- 분석 결과를 **이해하기 쉽고 보기 좋게 다양한 형태**로 표현
* R, Python, Tableau
## 3.6 폐기
- 더 이상 사용하지 않거나 허가된 사용 기한 이후 데이터 폐기
* 오버라이팅, 디가우싱 등 활용
# 4장 빅데이터 처리과정별 기술
## 4.1 수집 기술
* 로그수집기, 센싱, 크롤링, RSS 리더, 오픈 API
* ETL: 다양한 소스 → 공통 포맷 변환 및 적재
## 4.2 저장 기술
### 4.2.1 Hadoop
* HDFS + MapReduce
* 낮은 비용, 빠른 처리 속도
### 4.2.2 NoSQL
* 비정형 데이터 저장에 유리, 확장성 우수
* MongoDB, HBase, Cassandra 등
## 4.3 처리 기술
### 4.3.1 일괄 처리
* Hadoop MapReduce, Google MapReduce
### 4.3.2 실시간 처리
* IBM InfoSphere Streams, Twitter Storm
### 4.3.3 프로그래밍 언어 지원
* Google Sawzall, Hadoop Pig 등
## 4.4 분석 기술
### 4.4.1 텍스트마이닝
* 비정형 텍스트 분석, 연계성 파악
### 4.4.2 웹마이닝
* 웹에서 수집한 데이터 분석
### 4.4.3 기계학습
* 학습을 통한 예측/분류
### 4.4.4 감성분석
* 문장 의미 해석 및 긍정/부정 분류
# 5장 빅데이터 보안 개념

- 빅데이터 시스템 보안 이슈
## 5.1 개인정보 처리 과정별 보안 이슈
### 5.1.1 수집
- 데이터 수집 시 **개인 정보 포함 여부** 확인 필요, **유출 우려**
- 데이터 생성하는 주체의 **사전/사후 동의 필요**
- 수집되는 데이터에 대한 **접근 통제 필요**
### 5.1.2 저장
- 수집 데이터의 **안전한 저장 및 관리**
	- 암호화 저장
- 데이터 **등급 분류 및 필터링**
### 5.1.3 처리 및 분석
- **익명화된 데이터** 처리 및 분석
- **암호화된 데이터**의 처리
- **이용 목적 외의 처리 및 분석 방지**
### 5.1.4 표현(시각화)
- **개인정보를 침해할 수 있는 정보** 생성 위험이 있으므로 유의
- 분석된 정보의 무단 이용 X
### 5.1.5 폐기
- 데이터 활용 이후 **완전하게 폐기**되었는지 확인
- 폐기 방법
	- 논리적인  방법
	- 하드디스크 천공, 디가우싱 등 물리적인 방법
