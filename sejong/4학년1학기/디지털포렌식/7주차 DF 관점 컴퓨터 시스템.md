# 1. PC Teardown을 통한 컴퓨터 구조
## 1.1 컴퓨터 시스템의 구성 요소
### 1.1.1 컴퓨터 시스템과 생명체의 유사성
- 컴퓨터 시스템은 여러 구성 요소가 협력해 입력, 처리, 출력, 저장을 수행한다.
- 외형은 다양하지만 내부의 기본 동작 원리는 대부분 동일하다.
### 1.1.2 컴퓨터 시스템의 계층 구조
| 계층 | 역할 | 포렌식 의미 |
| --- | --- | --- |
| 하드웨어 | 연산, 저장, 입출력 수행 | 장치 구성과 저장 매체 흔적 확인 |
| 운영체제 | 자원 관리와 실행 환경 제공 | 로그, 레지스트리, 파일시스템 흔적 생성 |
| 애플리케이션 | 사용자의 작업 수행 | 실행 기록, 문서, 캐시, 설정 파일 생성 |
| 사용자 | 입력과 의사 결정 주체 | 계정별 행위 추적 대상 |
## 1.2 컴퓨터 시스템의 동작 원리
### 1.2.1 정보 처리의 기본 흐름
- **입력 → 처리 → 출력 및 저장**의 주기를 반복한다.
- 사용자가 입력 장치를 통해 명령을 내리면, 주기억장치(RAM)에 데이터가 적재되고 CPU가 이를 연산하여 결과를 출력 장치나 보조기억장치에 전달한다.
### 1.2.2 부팅 프로세스
- 전원 버튼을 누르면 전원공급장치(PSU)가 각 부품에 전력을 공급한다.
- 메인보드의 ROM에 저장된 **BIOS/UEFI**가 실행되어 하드웨어 이상 유무를 점검하는 **POST(Power-On Self-Test)** 과정을 거친다.
- 점검이 완료되면 보조기억장치(HDD/SSD)에 있는 운영체제(OS)의 핵심 부분(Kernel)을 주기억장치(RAM)로 불러와(Load) 시스템 제어권을 넘긴다.
### 1.2.3 CPU의 연산 과정
- CPU는 인출, 해독, 실행, 저장 사이클을 반복한다.
- 이 과정에서 메모리의 명령어를 읽고, 의미를 해석하고, 연산 결과를 다시 메모리나 레지스터에 저장한다.
# 2. DF 관점에서의 기억장치 분류
## 2.1 기억장치의 분류
### 2.1.1 기억장치의 개념과 역할
- 기억장치는 컴퓨터에서 **데이터와 명령어를 저장**하고 유지하는 핵심 구성 요소이다.
- CPU는 연산 처리를 위해 주기억장치에 저장된 데이터를 직접 읽고 처리한다.
- 보조기억장치에 저장된 데이터는 CPU가 직접 처리할 수 없으므로, 실행 시 반드시 주기억장치로 먼저 적재되어 처리된다.
- 전원 공급 여부와 저장 원리에 따라 여러 방식으로 분류된다.
### 2.1.2 전원 공급(휘발성)에 따른 분류
| 구분 | 특징 | 예시 | 포렌식 의미 |
| --- | --- | --- | --- |
| 휘발성 메모리 | 전원 차단 시 데이터 소실 | RAM | 라이브 포렌식에서 우선 수집 |
| 비휘발성 메모리 | 전원 차단 후에도 데이터 유지 | SSD, HDD | 이미징과 파일시스템 분석 대상 |
### 2.1.3 저장 원리에 따른 분류
- **반도체 저장매체**: 전자적인 방식으로 데이터를 저장한다. (예: RAM, ROM, 플래시 메모리 기반 SSD)
- **자기 저장매체**: 자성 물질과 자기장을 이용해 데이터를 기록한다. (예: HDD, 플로피디스크)
- **광학 저장매체**: 레이저 반사를 이용해 데이터를 읽고 쓴다. (예: CD, DVD, Blu-ray)
## 2.2 반도체를 이용한 저장매체
### 2.2.1 반도체 기반 저장매체의 특징
- 기계식 구동부가 없어 빠르고 조용하며 충격에 강하다.
- 발열과 전력 소모가 적어 노트북, 스마트폰, SSD 등에 널리 사용된다.
### 2.2.2 RAM (Random Access Memory)
- 전원 공급이 중단되면 데이터가 지워지는 대표적인 **휘발성** 메모리이다.
- 데이터의 읽기와 쓰기가 자유로우며, CPU의 주 작업 공간(주기억장치)으로 사용된다.
- **DRAM (Dynamic RAM)**: 일정 시간이 지나면 방전되므로 주기적인 재충전이 필요하다. 구조가 단순하고 대용량화에 유리하여 일반적인 PC의 메인 메모리로 사용된다.
- **SRAM (Static RAM)**: 전원이 공급되는 한 데이터가 계속 유지된다. 속도가 DRAM보다 훨씬 빠르지만 가격이 비싸 CPU 내부의 캐시(Cache) 메모리로 주로 사용된다.
- **활성화 시스템 분석**: 이와 같은 살아있는 시스템을 대상으로 메모리에 있는 데이터를 수집하는 것.
### 2.2.3 ROM (Read Only Memory)
- 전원이 차단되어도 데이터가 보존되는 **비휘발성** 메모리이다.
- 원칙적으로 데이터 읽기만 가능하며, 컴퓨터 부팅에 필수적인 시스템 펌웨어(BIOS)를 저장하는 데 사용된다.
- **종류**: 제조 시 데이터가 영구 기록되는 Mask ROM, 1회 쓰기가 가능한 PROM, 자외선으로 지우는 EPROM, 전기적으로 데이터를 쓰고 지울 수 있는 EEPROM으로 발전해 왔다.
### 2.2.4 플래시 메모리 (Flash Memory)
- EEPROM을 발전시킨 형태로, 전기적으로 데이터를 자유롭게 지우고 다시 쓸 수 있는 **비휘발성** 메모리이다.
- USB 플래시 드라이브, SD 카드, 스마트폰 내장 메모리 및 SSD(Solid State Drive)를 구성하는 핵심 소자로 널리 사용된다.

#### 2.2.4.1 NOR 플래시
- 빠른 속도와 소용량 특성을 가진다.
- CPU와 직접 통신할 수 있어 펌웨어 저장에 주로 사용된다.

#### 2.2.4.2 NAND 플래시
- 대용량 저장에 적합하며 블록 단위로 데이터를 전송한다.
- SSD, USB, SD 카드 등 저장장치에 주로 사용된다.
# 3. DF 관점에서의 스토리지와 인터페이스
## 3.1 반도체 저장매체 (SSD)
### 3.1.1 SSD(Solid State Drive)의 특징
- NAND 플래시 메모리 기반의 비휘발성 저장매체이다.
- 임의 접근 속도가 빠르고, 기계식 구동부가 없어 충격과 소음에 강하다.
- 현대 PC와 모바일 기기의 주력 저장매체로 사용된다.
### 3.1.2 NAND 플래시 셀 구조에 따른 분류
| 구분 | 저장 비트 | 특징 |
| --- | --- | --- |
| SLC | 1bit/cell | 빠르고 수명이 길지만 비쌈 |
| MLC | 2bit/cell | 용량과 가격의 균형 |
| TLC | 3bit/cell | 저렴하고 대용량에 유리하지만 수명과 속도는 낮음 |
### 3.1.3 수명 관리 기능 (디지털 포렌식 관점의 주의점)
| 기능 | 내용 | 포렌식 영향 |
| --- | --- | --- |
| Wear Leveling | 기록 위치를 분산해 셀 수명을 연장 | 물리 위치와 논리 주소가 단순히 대응되지 않을 수 있음 |
| TRIM | 삭제 블록을 SSD 컨트롤러에 알려 성능 저하를 방지 | 삭제 데이터 복구 가능성이 HDD보다 낮음 |

## 3.2 자기 저장매체(HDD)
### 3.2.1 HDD의 구조와 특징
- 자기 플래터(Magnetic Platter)에 데이터를 기록한다.
- 모터로 플래터를 회전시키고 헤드로 데이터를 읽고 쓴다.
- 대용량 저장에 유리하고 가격이 저렴하다.
- NAND 셀 수명 제한이 없다는 장점이 있다.
- SSD보다 속도가 느리고, 기계식 구조 때문에 충격에 취약하다.
- 소음과 발열이 발생할 수 있다.
## 3.3 광학 저장매체 및 저장장치 인터페이스
### 3.3.1 광학 저장매체 (Optical Storage Media)
- CD-ROM, DVD, Blu-Ray 등이 대표적이다.
- 레이저로 디스크 표면의 피트와 랜드를 읽고 쓰는 방식이다.
- 현재는 주로 보조 저장, 배포, 백업 용도로 제한적으로 사용된다.
### 3.3.2 저장장치 인터페이스 (연결 규격)
| 규격 | 특징 | 포렌식 의미 |
| --- | --- | --- |
| ATA / SATA | 병렬 ATA에서 직렬 SATA로 발전한 PC용 저장장치 연결 규격 | 내부 저장장치 연결 방식 확인 |
| SCSI / SAS | 서버·엔터프라이즈 환경에서 사용되는 고신뢰성 규격 | RAID, 서버 스토리지 분석 시 확인 |
| USB | 외장 저장장치와 주변기기 연결에 널리 사용 | 자료 반출, 외부 장치 연결 흔적 분석 |
## 3.4 스토리지 시스템 (RAID & 숨겨진 영역)
### 3.4.1 스토리지 시스템과 RAID의 개념
- **RAID (Redundant Array of Inexpensive Disks)**: 여러 개의 물리적인 하드 디스크(혹은 SSD)를 논리적으로 묶어 하나의 거대한 단일 스토리지처럼 운영하는 시스템 기술이다.
- 시스템의 데이터 입출력 성능을 극대화하거나, 동일한 데이터를 복제하여 결함 허용도(안정성)를 높이는 것이 주된 목적이다.
### 3.4.2 주요 RAID 레벨과 특징
| 레벨 | 방식 | 장점 | 한계 |
| --- | --- | --- | --- |
| RAID 0 | 스트라이핑 | 속도 향상 | 디스크 하나만 고장 나도 전체 데이터 손실 |
| RAID 1 | 미러링 | 복구 안정성 높음 | 실제 사용 가능 용량 감소 |
| RAID 5 | 스트라이핑 + 패리티 | 속도와 안정성 균형 | 최소 3개 디스크 필요 |
| RAID 1+0 | 미러링 + 스트라이핑 | 속도와 안정성 모두 확보 | 디스크 비용 증가 |
### 3.4.3 스토리지의 숨겨진 영역 (HPA & DCO)
- 악의적인 사용자나 범죄자가 중요한 증거 데이터를 은닉하기 위해 디스크의 숨겨진 영역(Hidden Area)을 악용할 수 있으므로, 포렌식 관점에서 이를 식별하는 것은 매우 중요하다.

#### 3.4.3.1 HPA
- HPA는 `Host Protected Area`이다.
- 디스크 끝부분에 예약된 숨겨진 영역이다.
- 존재 여부를 확인한 뒤 전체 디스크 이미징 범위에 포함해야 한다.

#### 3.4.3.2 DCO
- DCO는 `Device Configuration Overlay`이다.
- 디스크 크기나 기능을 제조사 수준에서 제한하는 영역이다.
- 실제 용량과 운영체제가 보고하는 용량의 차이를 확인해야 한다.

- 이미징 단계에서 HPA와 DCO를 점검하지 않으면 증거 영역이 누락될 수 있다.
