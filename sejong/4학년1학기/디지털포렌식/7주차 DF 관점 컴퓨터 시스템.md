# 1차시: PC Teardown을 통한 컴퓨터 구조
## 1. 컴퓨터 시스템의 구성 요소
### 1.1 컴퓨터 시스템과 생명체의 유사성
- 컴퓨터 시스템은 인간이 만든 인공적인 생명체와 유사한 구조를 가진다.
- 생명체가 여러 생체 기관의 협력과 소통으로 생명을 유지하듯, 컴퓨터 시스템도 여러 구성 요소가 유기적으로 상호작용하며 동작한다.
- 각 하드웨어의 역할은 생명체의 기관에 비유할 수 있다.
    - **CPU**: 뇌 역할 (연산 및 제어)
    - **메모리(RAM)**: 기억 기관 역할
    - **저장장치(HDD/SSD)**: 정보 저장 역할
    - **전원공급장치(PSU)**: 에너지 공급 역할
- 외형은 스마트폰, 노트북, 스마트워치 등 매우 다양하지만 내부의 기본 동작 원리는 대부분 동일하다.
### 1.2 컴퓨터 시스템의 계층 구조
- 컴퓨터 시스템은 크게 **하드웨어 → 운영체제(OS) → 애플리케이션 → 사용자**의 계층 구조로 구성된다.
- 이를 생명체에 비유하면 다음과 같다.
    - **하드웨어**: 생명체의 몸체
    - **운영체제(OS)**: 시스템을 동작하게 하는 영혼 (운영체제가 설치되어야 비로소 컴퓨터가 목적에 맞게 동작 가능)
    - **애플리케이션**: 생명체의 행동
    - **사용자**: 생명체의 의식
- **디지털 포렌식 관점의 중요성**: 사용자의 입력 및 출력 활동 과정에서 컴퓨터의 각 계층마다 수많은 기록(Log)과 흔적 데이터가 생성되며, 이는 디지털 포렌식 조사에서 사건을 재구성하는 핵심적인 단서가 된다.
## 2. 컴퓨터 시스템의 동작 원리
### 2.1 정보 처리의 기본 흐름
- **입력(Input) → 처리(Process) → 출력(Output) 및 저장(Storage)**의 주기를 반복한다.
- 사용자가 입력 장치를 통해 명령을 내리면, 주기억장치(RAM)에 데이터가 적재되고 CPU가 이를 연산하여 결과를 출력 장치나 보조기억장치에 전달한다.
### 2.2 부팅(Booting) 프로세스
- 전원 버튼을 누르면 전원공급장치(PSU)가 각 부품에 전력을 공급한다.
- 메인보드의 ROM에 저장된 **BIOS/UEFI**가 실행되어 하드웨어 이상 유무를 점검하는 **POST(Power-On Self-Test)** 과정을 거친다.
- 점검이 완료되면 보조기억장치(HDD/SSD)에 있는 운영체제(OS)의 핵심 부분(Kernel)을 주기억장치(RAM)로 불러와(Load) 시스템 제어권을 넘긴다.
### 2.3 CPU의 연산 과정
- CPU는 메모리에서 명령어를 가져오는 **인출(Fetch)**, 명령어의 의미를 해석하는 **해독(Decode)**, 실제 연산을 수행하는 **실행(Execute)**, 결과를 다시 메모리나 레지스터에 저장하는 저장(Store)의 사이클을 매우 빠른 속도로 반복하며 동작한다.
# 2차시: DF 관점에서의 기억장치 분류
## 1. 기억장치의 분류
### 1.1 기억장치의 개념과 역할
- 기억장치는 컴퓨터에서 데이터와 명령어를 저장하고 유지하는 핵심 구성 요소이다.
- CPU는 연산 처리를 위해 주기억장치에 저장된 데이터를 직접 읽고 처리한다.
- 보조기억장치에 저장된 데이터는 CPU가 직접 처리할 수 없으므로, 실행 시 반드시 주기억장치로 먼저 적재(Load)되어 처리된다.
- 전원 공급 여부에 따라 주기억장치(휘발성)와 보조기억장치(비휘발성)로 명확히 구분된다.
### 1.2 전원 공급(휘발성)에 따른 분류
- **휘발성 메모리 (Volatile Memory)**
    - 전원 공급이 차단되면 저장된 데이터가 모두 소실되는 특징을 가진다.
    - CPU와 가장 빈번하게 상호작용하며 빠른 속도로 데이터를 주고받는 작업 공간 역할을 한다.
    - **예시**: RAM (DDR 계열 등)
- **비휘발성 메모리 (Non-Volatile Memory)**
    - 전원이 차단된 후에도 내부의 데이터가 그대로 유지된다.
    - 데이터를 영구적으로, 장기 저장할 목적으로 사용된다.
    - **예시**: SSD (M.2, SATA 인터페이스), HDD 등
### 1.3 저장 원리에 따른 분류
- **반도체 저장매체**: 전자적인 방식을 사용하여 데이터를 저장하고 읽는다. 물리적인 움직임이 없어 속도가 빠르다. (예: RAM, ROM, 플래시 메모리 기반의 SSD)
- **자기 저장매체**: 자성 물질이 코팅된 플래터에 자기장을 이용하여 데이터를 기록하는 방식이다. (예: 하드디스크(HDD), 플로피디스크 등)
- **광학 저장매체**: 디스크 표면에 빛(레이저)을 쏘아 반사되는 정도를 이용해 데이터를 읽고 쓰는 방식이다. (예: CD, DVD, Blu-ray 등)
## 2. 반도체를 이용한 저장매체
### 2.1 반도체 기반 저장매체의 특징
- 기계적인 구동 부품(모터, 헤드 등)이 없어 데이터 처리 속도가 매우 빠르다.
- 물리적인 마찰이 없으므로 소음이 발생하지 않으며 발열과 전력 소모가 적다.
- 충격에 강하고 내구성이 뛰어나 노트북, 스마트폰 등 모바일 기기에 필수적으로 사용된다.
### 2.2 RAM (Random Access Memory)
- 전원 공급이 중단되면 데이터가 지워지는 대표적인 **휘발성** 메모리이다.
- 데이터의 읽기와 쓰기가 자유로우며, CPU의 주 작업 공간(주기억장치)으로 사용된다.
- **DRAM (Dynamic RAM)**: 일정 시간이 지나면 방전되므로 주기적인 재충전(Refresh)이 필요하다. 구조가 단순하고 대용량화에 유리하여 일반적인 PC의 메인 메모리로 사용된다.
- **SRAM (Static RAM)**: 전원이 공급되는 한 데이터가 계속 유지된다. 속도가 DRAM보다 훨씬 빠르지만 가격이 비싸 CPU 내부의 캐시(Cache) 메모리로 주로 사용된다.
- **활성화 시스템 분석**: 이와 같은 살아있는 시스템을 대상으로 메모리에 있는 데이터를 수집하는 것. 생명체가 죽기전 모든 것을 심문.
### 2.3 ROM (Read Only Memory)
- 전원이 차단되어도 데이터가 보존되는 **비휘발성** 메모리이다.
- 원칙적으로 데이터 읽기만 가능하며, 컴퓨터 부팅에 필수적인 시스템 펌웨어(BIOS)를 저장하는 데 사용된다.
- **종류**: 제조 시 데이터가 영구 기록되는 Mask ROM, 1회 쓰기가 가능한 PROM, 자외선으로 지우는 EPROM, 전기적으로 데이터를 쓰고 지울 수 있는 EEPROM으로 발전해 왔다.
### 2.4 플래시 메모리 (Flash Memory)
- 충격에 강하고 저전력 구동 가능
- EEPROM을 발전시킨 형태로, 전기적으로 데이터를 자유롭게 지우고 다시 쓸 수 있는 **비휘발성** 메모리이다.
- 데이터를 바이트 단위가 아닌 **블록(Block)** 단위로 읽고 쓰기 때문에 처리 속도가 매우 빠르다.
- USB 플래시 드라이브, SD 카드, 스마트폰 내장 메모리 및 SSD(Solid State Drive)를 구성하는 핵심 소자로 널리 사용된다.
- NOR 플래시
	- 빠른 속도, 소용량
	- CPU 직접 통신 가능 -> 주로 펌웨어 직접 저장
 - NAND 플래시
	- 대용량, 블록 단위 데이터 전송, 느린 속도
	- SSD, USB, SD 카드 등 저장장치 용
	- **SLC(Single Level Cell)**: 한 셀에 1비트를 저장하는 방식
		- 쓰기/지우기 수명은 10~15만번
		- 저장 용량 상대적으로 작음
	- **MLC(Multi-Level Cell)**: 한 셀에 2비트를 저장하는 방식
		- SLC보다 저장 용량 큼
		- 쓰기/지우기 수명 비교적 짧음
	- **TLC(Triple-Level Cell)**: 한 셀에 3비트를 저장하는 방식
		- MLC보다 저장 용량 큼
		- 쓰기/지우기 수명 더 짧음
# 3차시: DF 관점에서의 스토리지와 인터페이스
## 1. 반도체 저장매체 (SSD)
### 1.1 SSD(Solid State Drive)의 특징
- NAND 플래시 메모리를 기반으로 구성된다.
- 전원이 차단되어도 데이터가 유지되는 비휘발성 매체이다.
- RAM과 유사한 Random Access(임의 접근) 방식을 사용하여 데이터 접근 속도가 매우 빠르다.
- 모터나 디스크 같은 기계적 구동부가 없는 전자식 구조이므로 충격에 강하고, 소음과 발열이 없으며 전력 소비가 적다.
- **장점**: 데이터 처리 속도가 하드디스크 대비 압도적으로 빠르며, 가볍고, 충격 저항성이 뛰어나 모바일 기기 및 현대 PC의 주력 저장매체로 활용된다.
### 1.2 NAND 플래시 셀(Cell) 구조에 따른 분류
- 하나의 셀(저장 공간)에 몇 비트의 데이터를 저장하느냐에 따라 나뉜다.
- **SLC (Single-Level Cell)**: 1셀당 1비트 저장. 처리 속도가 가장 빠르고 수명이 길지만, 생산 단가가 비싸 주로 기업용 고성능 서버에 사용된다.
- **MLC (Multi-Level Cell)**: 1셀당 2비트 저장. 용량이 크며 SLC 대비 가격이 저렴하지만, 안정성과 수명이 약간 낮다.
- **TLC (Triple-Level Cell)**: 1셀당 3비트 저장. 대용량 구현이 가능하고 가격이 가장 저렴하여 일반 소비자용으로 널리 보급되었으나, 수명이 상대적으로 짧고 처리 속도가 느리다.
### 1.3 수명 관리 기능 (디지털 포렌식 관점의 주의점)
- **웨어 레벨링 (Wear Leveling)**: 특정 셀만 반복적으로 사용되어 수명이 다하는 것을 막기 위해, 데이터 기록 위치를 골고루 분산시켜 플래시 메모리의 전체 수명을 연장하는 기술이다.
- **TRIM 기능**: 운영체제에서 파일이 삭제되었을 때, 그 데이터 블록 정보를 SSD 컨트롤러에 미리 알려주어 실제 쓰기 작업 시 불필요한 삭제 과정을 생략해 성능 저하를 방지하는 명령어이다.
- **포렌식 조사 시 한계점**: TRIM 기능이 켜져 있는 SSD는 사용자가 파일을 삭제하면 실제 메모리 영역에서도 데이터를 물리적으로 소거할 확률이 높으므로, 하드디스크(HDD)에 비해 삭제된 데이터의 복구(Recovery)가 현저히 어렵다는 특징이 있다.

## 2. 자기 저장매체(HDD)
- 구성: 자기 플래터(Magnetic Platter)에 데이터 기록
- 구조: 기계식, 모터로 회전 (5400~15000 rpm), 헤드로 데이터 읽기/쓰기
- 특징
	- 대용량 데이터 저장에 유리
	- 가격 저렴
	- 속도는 SSD보다 느림
	- 기계식 구조로 충격에 취약, 소음·발열 있음
- 장점: 용량 크고 가격 저렴, NAND 셀 수명 제한 없음
- 단점: 속도 느림, 충격 취약, 소음/발열 존재
## 3. 광학 저장매체 및 저장장치 인터페이스
### 3.1 광학 저장매체 (Optical Storage Media)
- **종류**: CD-ROM, DVD, Blu-Ray 등
- **데이터 처리 방식**: 디스크 표면에 레이저를 쏘아 미세한 굴곡(피트와 랜드)을 읽고 쓰는 광학적 방식을 사용한다.
- **표준 규격**: ISO 9660 등의 국제 표준 파일 시스템 규격을 기반으로 제작되어, 다양한 운영체제와 하드웨어 간에 높은 호환성을 보장한다.
- **현재 용도**: 과거에는 주력 데이터 저장 및 배포 매체였으나, 현재는 대용량/고속 매체의 등장으로 인해 주로 보조 저장용이나 중요 데이터의 백업용으로 제한적으로 사용된다.
### 3.2 저장장치 인터페이스 (연결 규격)
- **ATA → SATA**: 과거 넓고 납작한 케이블을 쓰던 병렬(Parallel) 전송 방식에서, 얇은 케이블의 직렬(Serial) 전송 방식(SATA)으로 발전하였다. 현재 일반 PC용 스토리지 연결의 표준 규격으로 사용된다.
	- ATA(Advanced Technology Attachment): 하드디스크를 연결시키는 표준 인터페이스.
		- 현재 ATA-1부터 ATA/ATAPI-8까지 발전함
	- EIDE 인터페이스(ATA3)
		- 기존의 IDE 컨트롤러가 2개의 장치만 연결할 수 있던 점을 보완.
		- 하나의 IDE에서 더 많은 장치들을 연결할 수 있도록 설계
		- 각 채널은 다시 2개의 장치 연결을 위해 MS(Master/Slave) 방식 사용
	- SATA(Serial ATA): 직렬 전송 방식의 인터페이스
		- 최대 133 MBps 속도에 비해, SATA1은 150 MBps, SATA2는 300, SATA3은 600까지 지원.
- **SCSI → SAS**: 일반 PC보다 더 높은 신뢰성과 빠른 속도, 안정적인 데이터 전송이 필수적으로 요구되는 고성능 서버(Server) 및 엔터프라이즈급 스토리지에 주로 사용된다. 병렬 -> 직렬
- **USB (Universal Serial Bus)**:
    - 범용 직렬 통신 규격으로, 외장 하드디스크나 USB 메모리 등 다양한 주변 장치 연결에 쓰인다.
    - 뛰어난 이동성과 접근성 때문에 기밀 유출 사건 등에 자주 악용되며, 따라서 디지털 포렌식 조사 시 항상 최우선적인 수집 대상이 된다.
    - 소켓 내부의 플라스틱 색상(예: 파란색은 USB 3.0 이상)을 통해 직관적으로 지원 속도 및 타입을 구분할 수 있다.
## 4. 스토리지 시스템 (RAID & 숨겨진 영역)
### 4.1 스토리지 시스템과 RAID의 개념
- **RAID (Redundant Array of Inexpensive Disks)**: 여러 개의 물리적인 하드 디스크(혹은 SSD)를 논리적으로 묶어 하나의 거대한 단일 스토리지처럼 운영하는 시스템 기술이다.
- 시스템의 데이터 입출력 성능을 극대화하거나, 동일한 데이터를 복제하여 결함 허용도(안정성)를 높이는 것이 주된 목적이다.
### 4.2 주요 RAID 레벨과 특징
- **RAID 0 (스트라이핑, Striping)**
    - 데이터를 여러 디스크에 블록 단위로 쪼개어 동시 다발적으로 분산 저장한다.
    - 입출력 속도를 극대화할 수 있어 고속 데이터 처리에 유리하다.
    - **단점**: 데이터를 보호하는 기능이 없어, 묶여있는 디스크 중 단 하나라도 오류가 발생하면 전체 데이터를 완전히 잃게 되는 치명적인 단점이 있다 (안정성 취약).
- **RAID 1 (미러링, Mirroring)**
    - 동일한 데이터를 2개 이상의 디스크에 똑같이 복제(거울처럼 반사)하여 중복 저장한다.
    - 하나의 디스크가 물리적으로 고장 나더라도 다른 디스크에 온전한 원본 데이터가 그대로 남아있어 안정성과 복구 능력이 매우 뛰어나다.
    - **단점**: 전체 디스크 물리 용량의 절반밖에 사용하지 못하므로 스토리지 공간 효율성이 가장 떨어진다.
- RAID 5
	- 스트라이핑 + 패리티 -> 속도와 안정성 균형
	- 최소 3개 디스크 필요, 패리티로 오류 복구 가능
- RAID 1+0
	- RAID 1 + RAID 0 결합 → 속도와 안정성 모두 확보
	- 병렬 접근 가능, 서버 환경에서 많이 사용
### 4.3 스토리지의 숨겨진 영역 (HPA & DCO)
- 악의적인 사용자나 범죄자가 중요한 증거 데이터를 은닉하기 위해 디스크의 숨겨진 영역(Hidden Area)을 악용할 수 있으므로, 포렌식 관점에서 이를 식별하는 것은 매우 중요하다.
- **HPA (Host Protected Area)**
	- ATA-4 표준에서 추가됨, HDD에 의해 미리 예약된 영역
	- BIOS, UEFI 프로그램 등으로 접근 가능
    - 디스크의 끝부분에 위치하는 할당되지 않은 숨겨진 영역이다.
    - 주로 PC 제조사(OEM)가 시스템 복구 파티션이나 하드웨어 진단 도구 등을 안전하게 보관하기 위해 사용하며, 운영체제(OS)나 일반적인 파일 관리자에서는 이 영역의 용량이 인식되지 않는다.
- **DCO (Device Configuration Overlay)**
    - HPA보다 더 깊은 수준의 제어 영역으로, 디스크의 실제 크기나 지원 기능 집합을 사용자가 임의로 변경하지 못하도록 제조사에서 통제하는 영역이다.
- **포렌식 조사 시 대응**: 포렌식 조사관은 증거 수집(이미징) 단계에서 HPA와 DCO의 존재 여부를 반드시 점검해야 한다. 전용 포렌식 장비나 특수 소프트웨어를 이용해 해당 영역을 해제한 뒤 물리적인 디스크 전체를 빠짐없이 이미징해야 증거 누락을 막을 수 있다.
