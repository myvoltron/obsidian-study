# 1. 파일시스템 이해 ①
## 1.1 디스크 파티션
### 1.1.1 디스크 파티션 개념
- **파티션의 의미**
- 파티션은 하나의 큰 물리 디스크를 여러 개의 논리적 단위로 나누는 개념이다.
- 하나의 디스크를 여러 개의 디스크처럼 사용할 수 있다.
- **공간 분할 이유**
- 저장 용량이 커질수록 단일 공간으로는 관리가 어려워진다.
- 운영체제, 데이터, 백업 등을 구분하여 관리하기 위해 필요하다.
- **파티션 관리 프로그램**
- 디스크의 파티션 정보를 MBR/GPT 구조를 읽어 시각적으로 표시한다.
- 예시는 Windows의 `하드 디스크 파티션 만들기 및 포맷`이다.
### 1.1.2 파일시스템과 파티션의 관계
- 파일시스템은 파티션 위에 설치된다.
- 기본 구조는 다음과 같다.
```text
디스크 → 파티션 → 파일시스템 → 파일 구조
```
- 하나의 디스크에 여러 파티션을 만들 수 있다.
- 각 파티션마다 서로 다른 파일시스템을 사용할 수 있다.
## 1.2 MBR과 GPT
### 1.2.1 MBR
- **MBR의 의미**
- MBR은 `Master Boot Record`의 약자이다.
- 디스크 가장 앞부분 512바이트에 위치한다.
- 1983년에 도입된 전통적인 파티션 방식이다. (현재는 사라져 가고 있음.)
- **MBR의 역할**
- 디스크의 파티션 구조를 설명하는 지도 역할을 한다.
- 부팅 과정에서 운영체제가 위치한 파티션을 찾는 데 사용된다.
### 1.2.2 MBR 구조
MBR은 512바이트 레이아웃을 가진다.
- **Boot Code**
- 크기는 445바이트이다.
- 부팅 시 실행되는 코드이다.
- **Partition Table Entry**
- 4개의 파티션 엔트리가 존재한다.
- 각 파티션의 위치와 속성 정보를 저장한다.
- **Signature**
- 크기는 2바이트이다.
- MBR 식별용 값이다.
- 값은 `0x55AA`이다.
### 1.2.3 MBR 파티션 엔트리 주요 정보
각 MBR 파티션 엔트리는 16바이트이다.
- **부팅 가능 여부**
- Flag 값으로 부팅 가능 여부를 표시한다.
- `0x80`은 부팅 가능을 의미한다.
- `0x00`은 부팅 불가를 의미한다.
- **파티션 타입**
- NTFS, FAT 등 파일시스템 종류를 나타낸다.
- **파티션 위치 정보**
- 시작 CHS 주소를 포함한다.
- 끝 CHS 주소를 포함한다.
- CHS는 `Cylinder·Head·Sector`를 의미한다.
- 시작 LBA 주소를 포함한다.
- LBA는 선형 주소 방식이다.
- **섹터 수**
- 파티션 전체 크기 정보를 나타낸다.
### 1.2.4 MBR의 한계
- 최대 디스크 크기는 2TB이다.
- 최대 파티션 수는 4개이다.
- 파티션 정보가 디스크 앞부분 한 곳에만 저장된다.
- 파티션 정보가 손상되면 복구가 어렵다.
- 대용량 디스크 환경에 부적합하다.
### 1.2.5 GPT
- **GPT의 의미**
- GPT는 `GUID Partition Table`의 약자이다.
- MBR을 대체하는 최신 파티션 기법이다.
- UEFI 기반 컴퓨터에서 사용된다.
- GUID를 이용해 파티션을 고유하게 식별한다.
### 1.2.6 GPT의 장점
- 파티션 수 제한이 없다.
- 최대 9.4ZB까지 디스크 용량을 지원한다.
- 디스크 앞과 뒤 두 곳에 파티션 정보를 저장한다.
- 파티션 정보가 두 곳에 저장되므로 안정성이 향상된다.
- CRC 기반 무결성 검사를 지원한다.
- 백업 파티션 테이블이 존재하므로 복구가 가능하다.
## 1.3 파일시스템과 파일 탐색기
### 1.3.1 파일시스템의 핵심 역할
- 넓은 저장 공간을 체계적으로 관리한다.
- 사용자가 데이터를 직관적으로 접근할 수 있도록 지원한다.
- 데이터 저장 기능을 제공한다.
- 데이터 읽기 기능을 제공한다.
- 데이터 탐색 기능을 제공한다.
- 파일별 속성 정보를 효율적으로 관리한다.
### 1.3.2 메타데이터
- **메타데이터의 의미**
- 메타데이터는 파일의 속성 정보이다.
- **메타데이터에 포함되는 정보**
- 생성 시간
- 수정 시간
- 크기
- 접근 시간 등
- **메타데이터 관리 방식**
- 실제 파일 내용과 분리되어 관리된다.
- 파일시스템이 자동으로 유지·관리한다.
### 1.3.3 파일 구조 개념
- **Data Area**
- 파일의 실제 내용을 저장한다.
- **Meta Area**
- 파일의 속성 및 관리 정보를 저장한다.
- **파일시스템의 관리 방식**
- 파일시스템은 Data Area와 Meta Area 두 영역을 함께 관리한다.
### 1.3.4 파일 탐색기와 파일시스템의 상호작용
- 파일 탐색기를 실행하면 파일시스템에 디렉토리·파일 목록을 요청한다.
- 파일시스템은 각 파일의 메타데이터를 운영체제에 전달한다.
- 파일 탐색기는 정보를 정리해 사용자에게 화면으로 표시한다.
# 2. 파일시스템 이해 ②
## 2.1 스토리지
### 2.1.1 스토리지가 바라보는 두 가지 관점
- 스토리지는 물리적 관점과 논리적 관점으로 이해할 수 있다.
- 저장장치는 두 가지 시각으로 이해 가능하다.
### 2.1.2 물리적 관점
- **의미**
- 물리적 관점은 `Element-level` 관점이다.
- 실제 저장매체의 구조를 기준으로 보는 관점이다.
- **특징**
- 섹터 단위로 0과 1의 데이터가 저장된다.
- 시스템 내부 동작 기반의 구조이다.
### 2.1.3 논리적 관점
- **의미**
- 논리적 관점은 `Top-down` 관점이다.
- 파일 탐색기에서 보이는 구조를 기준으로 보는 관점이다.
- **특징**
- 디렉토리 형태로 표현된다.
- 폴더 형태로 표현된다.
- 파일 형태로 표현된다.
- 사람이 이해하기 쉬운 표현이다.
### 2.1.4 물리적 구조와 논리적 구조의 차이
- 논리적 구조는 사람을 위한 표현이다.
- 물리적 구조는 시스템 내부 동작 기반이다.
## 2.2 섹터와 클러스터
### 2.2.1 MBR과 섹터의 개념
- **MBR**
- MBR은 `Master Boot Record`의 약자이다.
- 크기는 512바이트이다.
- 디스크 파티션 정보를 담은 영역이다.
- **섹터**
- 섹터는 `Sector`이다.
- 저장장치의 물리적 최소 단위이다.
- 크기는 512바이트이다.
- **MBR과 섹터의 관계**
- MBR은 섹터 하나 전체에 저장된다.
- MBR은 디스크의 가장 앞부분 섹터를 사용한다.
### 2.2.2 클러스터의 개념
- **클러스터의 의미**
- 클러스터는 `Cluster`이다.
- 할당 단위라고도 한다.
- 파일시스템이 데이터를 관리하기 위해 여러 개의 섹터를 묶어 만든 논리적 단위이다.
- **클러스터 크기**
- 포맷 시 선택하는 `할당 단위 크기`가 클러스터 크기이다.
- **파일 저장 방식**
- 파일은 반드시 클러스터 단위로 저장된다.
- 파일 크기와 무관하게 최소 1클러스터를 차지한다.
### 2.2.3 클러스터 크기에 따른 특성
- **작은 클러스터**
- 장점은 공간 낭비가 감소한다는 것이다.
- 단점은 관리 테이블이 증가한다는 것이다.
- 단점은 성능 저하 가능성이 있다는 것이다.
- **큰 클러스터**
- 장점은 관리가 용이하다는 것이다.
- 장점은 상대적으로 빠른 접근이 가능하다는 것이다.
- 단점은 작은 파일 저장 시 공간 낭비가 증가한다는 것이다.
- **클러스터 크기 선택**
- 클러스터 크기 선택에는 장단점의 균형이 필요하다.
## 2.3 슬랙
### 2.3.1 슬랙 공간
- **정의**
- 파일 크기가 클러스터 크기보다 작으면 남는 공간이 발생한다.
- 이 남는 영역을 슬랙 공간이라고 한다.
- ram 슬랙과 file 슬랙으로 나뉜다. 
```text
파일 크기 < 클러스터 크기 → 남는 공간 발생
```
- **특징**
- 파일에 할당되었으나 실제로 사용되지 않은 공간이다.
- **의미**
- 파일시스템 관점에서는 불가피한 공간 낭비이다.
- 디지털 포렌식 관점에서는 과거 데이터가 남을 수 있는 중요 분석 영역이다.
# 3. FAT 파일시스템
## 3.1 FAT
### 3.1.1 FAT 파일시스템 이해의 필요성
- 파일시스템은 스토리지 내 데이터를 저장·관리·탐색하는 방식을 결정한다.
- FAT 구조를 이해하면 파일 탐색기의 동작 원리가 자연스럽게 연결된다.
- FAT 구조를 이해하면 디지털 포렌식 도구의 분석 방식이 자연스럽게 연결된다.
- FAT는 NTFS 이전 세대 파일시스템이다.
- FAT는 파일시스템 개념을 이해하기에 가장 기본적이고 직관적인 모델이다.
### 3.1.2 FAT의 의미
- **FAT의 약자**
- FAT는 `File Allocation Table`의 약자이다.
- **FAT의 역할**
- 모든 클러스터의 상태와 연결 정보를 기록한 장부이다.
- **FAT의 특징**
- 파일이 여러 클러스터에 흩어져 있어도 다음 클러스터 번호 정보만으로 끝까지 추적할 수 있다.
- **FAT의 구조적 특징**
- 연결 리스트 방식으로 파일 조각을 관리한다.
- 연결 리스트는 `Linked List` 방식이다.
## 3.2 FAT 파일시스템 구성
### 3.2.1 시스템 영역
FAT 파일시스템의 시스템 영역에는 다음 요소가 포함된다.
- **Boot Sector / Reserved Sector**
- 부팅 및 파일시스템 관리와 관련된 영역이다.
- **FAT 테이블**
- 각 클러스터의 상태와 연결 정보를 기록한다.
- **Root Directory**
- 루트 디렉터리 정보를 저장한다.
### 3.2.2 데이터 영역
- 실제 파일 데이터가 저장되는 클러스터들의 연속된 배열이다.
## 3.3 FAT의 기본 동작 원리
### 3.3.1 저장 관리 단위
- FAT의 저장 관리 단위는 클러스터이다.
### 3.3.2 파일 저장 방식
- 파일 크기가 클러스터 크기보다 크면 여러 클러스터에 나누어 저장된다.
```text
파일 크기 > 클러스터 크기 → 여러 클러스터에 나누어 저장
```
### 3.3.3 FAT 테이블의 역할
- FAT 테이블은 각 클러스터에 대해 다음 클러스터 번호를 기록한다.
- FAT 테이블은 각 클러스터에 대해 `End`를 기록할 수 있다.
- 다음 클러스터 번호는 파일이 이어지는 위치를 의미한다.
- `End`는 파일의 끝을 의미한다.
### 3.3.4 파일 읽기 과정
- FAT 테이블을 따라 이동하며 데이터에 접근한다.
- 예시는 다음과 같다.
```text
2 → 3 → 6 → End
```
- 위 구조에서는 2번 클러스터에서 시작한다.
- 다음으로 3번 클러스터로 이동한다.
- 다음으로 6번 클러스터로 이동한다.
- `End`에 도달하면 파일 읽기가 끝난다.
