# 1차시: 활성화 시스템 분석의 이해
## 1. 라이브 포렌식의 중요성
라이브 포렌식은 구동 중인 시스템에서 데이터를 수집하고 분석하는 기술이다. 시스템이 켜져 있는 상태에서만 확인 가능한 정보를 확보할 수 있기 때문에, 침해 사고 현장에서는 중요한 분석 방식이다.
## 2. 라이브 포렌식의 개념과 필요성
### 2.1 개념과 필요성
- 구동 중인 시스템에서 데이터를 수집하고 분석하는 기술이다.
- 사건 현장에서 시스템을 중단하지 않고 실시간 정보를 확보할 수 있다.
- 공격 흔적이나 증거가 훼손되기 전에 확보할 수 있다.
- 기존의 `Non-Volatile Storage` 기반 조사와 달리 활성 시스템 정보까지 포함한다.
### 2.2 라이브 포렌식이 필요한 이유
- 시스템이 꺼지면 사라지는 정보를 수집할 수 있다.
- 현재 실행 중인 프로세스와 네트워크 연결 상태를 확인할 수 있다.
- 공격자가 현재 어떤 행위를 하고 있는지 실시간으로 파악할 수 있다.
- 메모리, 세션, 네트워크, 프로세스 등 디스크에 남지 않을 수 있는 정보를 확보할 수 있다.
- 사건 당시의 시스템 상태를 최대한 보존해 분석할 수 있다.
## 3. 휘발성 데이터
휘발성 데이터는 전원이 차단되면 소멸되는 임시 데이터이다. 라이브 포렌식에서 가장 먼저 수집해야 하는 핵심 대상이다.
### 3.1 정의
- 전원 차단 시 소멸되는 임시 데이터이다.
- 시스템이 실행 중일 때만 존재한다.
- 재부팅 또는 종료 시 사라질 수 있다.
### 3.2 특성
- 실시간으로 변동한다.
- 수집 중에도 값이 변화할 수 있다.
- 공격 흔적을 포함할 가능성이 높다.
- 시스템 행위 분석에 중요하다.
### 3.3 중요성
- 공격 흔적 확보에 사용된다.
- 시스템 행위 분석에 사용된다.
- 사고 당시의 상태를 복원하는 데 사용된다.
- 디스크 분석만으로는 확인하기 어려운 정보를 제공한다.
### 3.4 예시
- RAM에서 실행 중인 프로세스
- 네트워크 연결 정보
    - IP
    - 포트
    - 세션
- 사용자 세션 정보
- CPU 실시간 사용 현황
- 메모리 실시간 사용 현황
## 4. 라이브 포렌식 수행 시 주의점
라이브 포렌식은 구동 중인 시스템을 대상으로 하기 때문에, 증거 수집 행위 자체가 시스템 상태를 변경할 수 있다.
### 4.1 데이터 훼손 가능성
- 수집 소프트웨어 실행 자체가 데이터를 훼손할 수 있다.
- 도구 실행으로 인해 메모리, 프로세스, 로그 등이 변경될 수 있다.
- 수집 과정에서 새로운 흔적이 생성될 수 있다.
### 4.2 시스템 성능 영향 최소화
- 데이터 수집 시 시스템 성능 영향을 최소화해야 한다.
- 과도한 도구 실행은 시스템 상태를 바꿀 수 있다.
- 시스템 부하가 증가하면 기존 프로세스나 네트워크 상태가 변할 수 있다.
### 4.3 절차와 도구의 신중한 사용
- 정확한 데이터 수집을 위해 절차를 신중하게 적용해야 한다.
- 상황에 맞는 도구를 선택해야 한다.
- 수집 순서와 명령 실행 기록을 남기는 것이 중요하다.
## 5. 라이브 포렌식의 장점
- 일반 포렌식보다 더 많은 유용 정보를 확보할 수 있다.
- 실시간 시스템 상태를 관찰할 수 있다.
- 실시간 시스템 행위를 관찰할 수 있다.
- 즉각적 대응에 활용할 수 있다.
- 증거 수집에 필수적인 정보를 확보할 수 있다.
## 6. 라이브 포렌식의 주요 도구 및 기술
라이브 포렌식에서는 메모리 덤프, 메모리 덤프 분석 도구, 시스템 정보 확인 도구, 네트워크 분석 도구 등이 활용된다.
## 7. 메모리 덤프
### 7.1 정의
- 구동 중인 시스템의 메모리 내용을 그대로 저장하는 작업이다.
- RAM에 존재하는 휘발성 데이터를 파일 형태로 확보하는 방식이다.
### 7.2 목적
- 사건 현장의 사진을 촬영하는 것과 유사한 역할이다.
- 휘발성 데이터를 확보하기 위해 사용한다.
- 실행 중인 프로세스, 네트워크 연결, 사용자 세션 등의 정보를 보존할 수 있다.
### 7.3 수집 도구
- `DumpIt`
- `Microsoft Crash Dump`
## 8. 메모리 덤프 분석 도구
### 8.1 Volatility
- 오픈소스 도구이다.
- 메모리 덤프 분석에 사용된다.
- 메모리 덤프에서 다양한 정보를 추출할 수 있다.
### 8.2 Rekall
- 구글에서 개발한 오픈소스 도구이다.
- Volatility와 동일하게 메모리 덤프 분석을 목적으로 한다.
- 메모리 내 시스템 정보와 행위 흔적을 분석하는 데 사용된다.
## 9. 시스템 정보 및 실시간 행위 확인 도구
### 9.1 Sysinternals Suite
- 실행 프로세스를 확인할 수 있다.
- 자원 사용 현황을 확인할 수 있다.
- 네트워크 연결 정보를 확인할 수 있다.
- 시스템의 실시간 행위 분석에 활용된다.
### 9.2 Wireshark
- 네트워크 패킷 분석 도구이다.
- 네트워크 통신을 심층적으로 관찰할 수 있다.
- 전체적인 네트워크 흐름을 거시적으로 파악할 수 있다.
## 10. 라이브 포렌식 특징
- 구동 중인 시스템에서 실시간 데이터를 수집한다.
- 휘발성 데이터를 확보한다.
- 공격 흔적을 분석한다.
- 시스템 행위를 분석한다.
- 수집 중 데이터가 변동될 수 있다.
- 분석 도구를 통해 정보를 추출한다.
## 11. 라이브 포렌식 절차
라이브 포렌식은 시스템 상태 확인에서 시작해 메모리 이미지 획득, 시스템 기본 정보 수집, 비휘발성 데이터 수집으로 이어진다.
## 12. 시스템 상태 확인
### 12.1 전원 상태 판단
- 시스템의 전원 `ON/OFF` 여부를 판단한다.
- 전원 상태에 따라 수집 절차가 달라진다.
### 12.2 전원 OFF 상태
- 시스템이 꺼져 있다면 비휘발성 데이터 이미지를 수집한다.
- 디스크 이미지, 파일 시스템 메타데이터, 로그 등이 주요 대상이다.
### 12.3 전원 ON 상태
- 시스템이 켜져 있다면 라이브 포렌식을 진행한다.
- 휘발성 데이터가 사라지기 전에 우선 수집한다.
## 13. 메모리 이미지 획득
메모리 이미지 획득은 휘발성 데이터를 수집하는 단계이다.
### 13.1 수집 대상
- 물리 메모리
- 가상 메모리 파일
- 시간 민감 데이터
### 13.2 수집 원칙
- 시간 민감 데이터를 우선 확보한다.
- 사라지기 쉬운 데이터부터 수집한다.
- 시스템 상태를 최대한 변경하지 않도록 주의한다.
### 13.3 도구
- `DumpIt`
- `FTK Imager`
## 14. 시스템 기본 정보 / 프로세스 / 네트워크 수집
### 14.1 수집 대상
- 실행 중인 프로세스
- 사용자 세션
- 네트워크 연결 정보
### 14.2 도구
- `Sysinternals Suite`
- `Wireshark`
## 15. 비휘발성 데이터 수집
비휘발성 데이터 수집은 선택적으로 수행된다.
### 15.1 수집 대상
- 파일 시스템 메타데이터
- 운영체제 로그
- 실행 파일
### 15.2 도구
- `FTK Imager` 등
## 16. 라이브 포렌식 핵심 원칙
### 16.1 시간 민감도 기반 우선순위 적용
- 먼저 사라지는 데이터를 우선 수집한다.
- 장기 저장 데이터는 이후 수집한다.
- 휘발성 데이터 → 비휘발성 데이터 순서로 접근한다.
### 16.2 적합 도구 활용
- 각 단계별로 적합한 도구를 사용한다.
- 도구 선택을 통해 증거 확보를 효율화한다.
- 수집 대상과 도구를 명확하게 매핑해야 한다.
## 17. 활용 포인트
- 사건 현장 상황에 따라 유연하게 적용한다.
- 수집 단계와 도구 매핑을 통해 체계적 분석이 가능하다.
- 실시간 대응과 증거 보존을 동시에 고려해야 한다.

# 2차시: 활성화 시스템 도구 실습
## 1. 휘발성 데이터, 비휘발성 데이터
활성화 시스템 분석에서는 데이터가 전원 상태에 따라 유지되는지 여부를 기준으로 휘발성 데이터와 비휘발성 데이터를 구분한다.
## 2. 휘발성 데이터
휘발성 데이터는 전원이 켜져 있을 때만 존재하고, 재부팅 시 사라지는 데이터이다.
### 2.1 정의
- 전원이 켜져 있을 때만 존재한다.
- 재부팅 시 사라진다.
- 종료 또는 전원 차단 시 소멸될 수 있다.
### 2.2 수집 대상
- 시스템 시간
- 프로세스 리스트
- 서비스 리스트
- 로그인 사용자 정보
- 클립보드 내용
- 네트워크 연결 상태
- 열린 파일 정보
### 2.3 특징
- 사건 당시의 실시간 정보를 담고 있다.
- 즉시 수집해야 한다.
- 수집이 지연되면 데이터가 사라지거나 변경될 수 있다.
## 3. 비휘발성 데이터
비휘발성 데이터는 전원이 꺼져도 유지되는 데이터이다.
### 3.1 정의
- 전원이 꺼져도 유지되는 데이터이다.
- 디스크나 저장 매체에 남아 있는 데이터가 중심이다.
### 3.2 수집 대상
- 파일 시스템 메타데이터
- 이벤트 로그
- 레지스트리
- 디스크 정보
- 브라우저 기록
- 응용 프로그램 기록
### 3.3 특징
- 사건 추적이 가능하다.
- 선별 수집이 가능하다.
- 휘발성 데이터보다 상대적으로 보존성이 높다.
## 4. 메모리 포렌식
활성화 시스템 도구 실습에서는 윈도우 기본 명령어와 Sysinternals 도구를 활용해 라이브 포렌식 정보를 수집하는 방법을 다룬다.
## 5. 윈도우 기본 명령어 활용
윈도우 기본 명령어는 별도 도구 없이 시스템 상태를 빠르게 확인하는 데 사용된다.
## 6. 시스템 정보 수집
### 6.1 현재 날짜 / 시간 확인
```cmd
date /t
time /t
```
- 사건 시점의 기준 시간을 확인한다.
- 시스템 시간이 실제 시간과 일치하는지 확인하는 데 사용된다.
- 이후 수집되는 로그와 아티팩트의 시간 기준을 맞추는 데 중요하다.
### 6.2 시스템 구동 시간 확인
```cmd
systeminfo
```
- 마지막 재부팅 시점을 확인할 수 있다.
- 시스템이 언제부터 켜져 있었는지 확인할 수 있다.
- 침해 사고 발생 시점과 시스템 재부팅 여부를 비교할 수 있다.
## 7. 사용자 계정 및 로그인 정보
### 7.1 계정 목록 확인
```cmd
net users
```
- 시스템에 존재하는 사용자 계정 목록을 확인한다.
- 의심 계정 또는 비인가 계정을 찾는 데 활용할 수 있다.
### 7.2 현재 로그인 사용자 확인
```cmd
psloggedon
```
- 현재 로그인한 사용자를 확인한다.
- Sysinternals 도구에 포함된 명령이다.
- 로컬 또는 원격 로그인 사용자를 확인하는 데 활용할 수 있다.
## 8. 프로세스 / 서비스 / 파일 / 네트워크 정보
### 8.1 네트워크 상태 확인
```cmd
netstat -ano
```
- 열린 포트를 확인한다.
- 네트워크 연결 상태를 확인한다.
- 프로세스 ID와 네트워크 연결을 매핑할 수 있다.
- 의심스러운 외부 연결을 추적하는 데 활용된다.
### 8.2 열려 있는 파일 확인
```cmd
openfiles
net file
psfile
```
- 실시간으로 접근 중인 파일을 확인한다.
- 열린 파일 정보를 확인한다.
- 파일 접근 흔적과 사용자 행위를 분석하는 데 활용된다.
- `psfile`은 Sysinternals 도구에 포함된 명령이다.
## 9. WMIC 명령어
WMIC는 윈도우 시스템의 다양한 정보를 명령어 기반으로 조회할 수 있는 도구이다. 시스템, 계정, 프로세스, 서비스, 네트워크, 설치 프로그램 등의 정보를 수집할 수 있다.
## 10. WMIC 명령어 활용: 심층 정보 수집
### 10.1 시스템 정보 수집
WMIC를 통해 다음 시스템 정보를 확인할 수 있다.
- 시스템 이름
- BIOS 정보
- CPU 정보
- 논리적 디스크 정보
- 메모리 정보
- USB 장치 정보
### 10.2 계정 / 프로세스 / 서비스 정보 수집
- 계정 정보를 확인할 수 있다.
- 프로세스 정보를 확인할 수 있다.
- 서비스 정보를 확인할 수 있다.
- 필터링과 조건문을 활용할 수 있다.
- 특정 프로세스를 선별할 수 있다.
- 활성화된 계정을 선별할 수 있다.
### 10.3 네트워크 및 설치 프로그램 정보 수집
- 네트워크 설정 정보를 획득할 수 있다.
- IP 정보를 획득할 수 있다.
- 설치된 프로그램 정보를 획득할 수 있다.
### 10.4 SQL 유사 구조 활용
- WMIC는 SQL과 유사한 구조를 활용할 수 있다.
- 다양한 속성을 조합해 맞춤 정보를 추출할 수 있다.
- 조건 기반 조회가 가능하다.
- 반복적인 정보 수집을 자동화하는 데 활용할 수 있다.
## 11. Sysinternals Suite
Sysinternals Suite는 윈도우 시스템 분석과 관리에 유용한 도구 모음이다. 라이브 포렌식에서는 프로세스, 네트워크, 파일, 레지스트리, 계정, 서비스, 메모리 분석 등에 활용된다.
## 12. GUI vs 명령어 기반 도구 비교
### 12.1 GUI 기반 도구
- 직관적인 시각화를 제공한다.
- 사용자가 정보를 눈으로 빠르게 파악하기 쉽다.
- 실시간 상태 확인에 편리하다.
### 12.2 명령어 기반 도구
- 세부 옵션을 지정할 수 있다.
- 스크립트 자동화가 가능하다.
- 반복적인 수집 작업에 효율적이다.
- 대량 시스템 분석 또는 정형화된 수집 절차에 적합하다.
## 13. Sysinternals 주요 도구 및 기능
### 13.1 네트워크 분석 도구
#### TCPView
- 네트워크 연결 상태를 확인한다.

#### PsPing
- 네트워크 연결 상태를 확인한다.
- 지연 시간이나 연결 가능성 확인에 활용된다.
### 13.2 파일 / 레지스트리 분석 도구
#### ProcMon
- 파일 접근을 모니터링한다.
- 레지스트리 접근을 모니터링한다.
- 프로세스의 실시간 행위를 확인하는 데 유용하다.
#### AccessChk
- 파일 권한을 확인한다.
- 객체 권한을 확인한다.
- 권한 설정 이상 여부를 분석하는 데 활용된다.
### 13.3 계정 / 서비스 분석 도구
#### PsExec
- 원격 실행에 사용된다.

#### PsService
- 서비스 정보를 확인한다.
- 서비스 상태를 확인한다.
- 의심 서비스 분석에 활용된다.

- 시작 프로그램, 서비스, 드라이버 등 자동 실행 항목을 분석한다.
- 악성코드 지속성 분석에 유용하다.
### 13.4 메모리 분석 도구
#### RAMMap
- 메모리 사용 현황을 분석한다.
- RAM 사용 상태를 확인하는 데 활용된다.
#### VMMap
- 프로세스별 가상 메모리 사용 현황을 분석한다.
- 프로세스 메모리 구조를 확인하는 데 활용된다.

# 3차시: 증거물을 이용한 실전 분석
## 1. 메모리 포렌식
증거물을 이용한 실전 분석에서는 메모리 덤프와 Volatility 프레임워크를 활용해 레지스트리 정보를 추출하고, 사용자 계정 및 시스템 정보를 분석하는 방법을 다룬다.
## 2. 레지스트리의 역할과 분석 필요성
레지스트리는 시스템과 사용자 활동을 분석할 수 있는 핵심 아티팩트이다. 활성 시스템 분석에서는 메모리 덤프에 존재하는 레지스트리 정보를 추출해 실전 분석에 활용할 수 있다.
### 2.1 레지스트리의 역할
- 사용자 설정을 저장한다.
- 시스템 환경 정보를 저장한다.
- 하드웨어 설치 이력을 보존한다.
- 소프트웨어 설치 이력을 보존한다.
- 로그온 기록을 추적할 수 있다.
- USB 사용 기록을 추적할 수 있다.
- 최근 실행 기록을 추적할 수 있다.
### 2.2 분석 필요성
- 시스템 활동을 분석할 수 있다.
- 사용자 활동을 분석할 수 있다.
- 디지털 블랙박스 역할을 한다.
- 침해 사고 발생 시 사용자의 행위와 시스템 변경 흔적을 확인할 수 있다.
## 3. Volatility 프레임워크 개요
Volatility는 메모리 덤프 분석을 위한 오픈소스 프레임워크이다. 다양한 플러그인을 통합해 메모리에서 여러 종류의 정보를 추출할 수 있다.
### 3.1 특징
- 메모리 덤프 분석용 오픈소스 프레임워크이다.
- 다양한 플러그인을 통합해 사용할 수 있다.
- 다중 정보 추출이 가능하다.
- GitHub에서 다운로드할 수 있다.
### 3.2 명령어 구조
```bash
volatility -f [덤프파일] --profile=[OS버전] [플러그인] -K [레지스트리키]
```
### 3.3 명령어 구성 요소
- `-f [덤프파일]`
    - 분석할 메모리 덤프 파일을 지정한다.
- `--profile=[OS버전]`
    - 분석 대상 운영체제 버전을 지정한다.
- `[플러그인]`
    - 실행할 Volatility 플러그인을 지정한다.
- `-K [레지스트리키]`
    - 추출하거나 분석할 레지스트리 키를 지정한다.
## 4. Volatility 활용 예시
### 4.1 레지스트리 키 추출 실습
Volatility를 사용하면 메모리 덤프에서 레지스트리 키를 추출할 수 있다.
### 4.2 Volatile Environment
- 현재 로그인한 사용자 환경 정보를 확인할 수 있다.
- 사용자 세션의 환경 변수와 관련된 정보를 분석하는 데 활용된다.
### 4.3 Profile List
- 시스템에 존재하는 사용자 계정 정보를 확인할 수 있다.
- 사용자 프로파일 목록을 통해 계정 흔적을 분석할 수 있다.
### 4.4 SID 분석
- SID를 분석해 사용자 계정을 식별할 수 있다.
- 계정 상태를 확인할 수 있다.
- 사용자 계정과 레지스트리 경로를 연결하는 데 활용된다.
### 4.5 덤프 파일 예시
```text
win7.mem
```
- 실습에서 사용할 수 있는 메모리 덤프 파일 예시이다.
## 5. 주요 레지스트리 경로
메모리 포렌식에서는 레지스트리의 주요 하이브와 경로를 이해하는 것이 중요하다.
## 6. HKEY_CURRENT_USER: HKCU
- 현재 로그인한 사용자의 환경 설정을 저장한다.
- 데스크탑 설정을 포함한다.
- 네트워크 설정을 포함한다.
- 애플리케이션 설정을 포함한다.
- 현재 사용자 행위 분석에 활용된다.
## 7. HKEY_USERS: HKU
- 시스템 내 모든 사용자 환경 설정을 저장한다.
- HKCU는 HKU 중 일부와 연결된다.
- 여러 사용자 계정의 환경 설정과 흔적을 확인하는 데 사용된다.
## 8. HKEY_LOCAL_MACHINE: HKLM
- 시스템 하드웨어 정보를 저장한다.
- 드라이버 설정을 저장한다.
- 부팅 구성을 포함한다.
- `BCD` 정보를 포함한다.
- `SAM` 정보를 포함한다.
- `SECURITY` 정보를 포함한다.
- `SOFTWARE` 정보를 포함한다.
- `SYSTEM` 정보를 포함한다.
## 9. HKEY_CURRENT_CONFIG: HKCC
- 현재 하드웨어 프로파일 정보를 저장한다.
- Volatile 데이터 영역이다.
- 시스템 부팅 시 동적으로 생성된다.
- 현재 시스템 하드웨어 상태를 확인하는 데 활용된다.
## 10. 레지스트리 구조 및 저장 방식
레지스트리는 Hive 파일 구조와 Key-Value 구조를 기반으로 데이터를 저장한다. 메모리 내 동적 정보와 파일 형태의 영구 데이터를 모두 포함할 수 있다.
### 10.1 Hive 파일 구조
- Hive 파일 구조를 가진다.
- `ntuser.dat` 등이 대표적인 Hive 파일이다.
- 사용자별 설정과 환경 정보가 저장될 수 있다.
### 10.2 Key-Value 형태 저장
- 레지스트리는 Key-Value 형태로 데이터를 저장한다.
- Key는 경로 또는 항목 역할을 한다.
- Value는 실제 데이터 또는 설정값을 담는다.
### 10.3 메모리 내 동적 정보와 영구 데이터
- 메모리 내 동적 정보를 포함한다.
- 파일 형태의 영구 데이터를 포함한다.
- 현재 실행 중인 시스템 상태와 저장된 설정 정보를 함께 분석할 수 있다.
## 11. 분석 대상 예시

- 사용자 계정 정보를 분석할 수 있다.
