# 1. 활성화 시스템 도구 실습
## 1.1 수집 대상 정리
### 1.1.1 휘발성 / 비휘발성 데이터
| 구분 | 수집 대상 | 포렌식 의미 |
| --- | --- | --- |
| 휘발성 데이터 | 시스템 시간, 프로세스 리스트, 서비스 리스트, 로그인 사용자 정보, 클립보드, 네트워크 연결, 열린 파일 | 전원 차단 전에 우선 수집해야 하는 실시간 정보 |
| 비휘발성 데이터 | 파일 시스템 메타데이터, 이벤트 로그, 레지스트리, 디스크 정보, 브라우저 기록, 응용 프로그램 기록 | 전원 차단 후에도 남아 사건 추적에 활용되는 정보 |

## 1.2 윈도우 기본 명령어
### 1.2.1 라이브 포렌식 기본 수집
| 목적 | 명령어 | 확인 내용 |
| --- | --- | --- |
| 현재 날짜 / 시간 확인 | `date /t`, `time /t` | 사건 시점 기준 시간, 시스템 시간 일치 여부 |
| 시스템 구동 시간 확인 | `systeminfo` | 마지막 재부팅 시점, 시스템 가동 시간 |
| 계정 목록 확인 | `net users` | 사용자 계정 목록, 의심 계정 여부 |
| 현재 로그인 사용자 확인 | `psloggedon` | 로컬 / 원격 로그인 사용자 |
| 네트워크 상태 확인 | `netstat -ano` | 열린 포트, 네트워크 연결, PID 매핑 |
| 열린 파일 확인 | `openfiles`, `net file`, `psfile` | 실시간 접근 파일과 사용자 행위 흔적 |

## 1.3 WMIC 명령어
### 1.3.1 WMIC 활용 범위
| 범주 | 확인 가능한 정보 | 활용 포인트 |
| --- | --- | --- |
| 시스템 정보 | 시스템 이름, BIOS, CPU, 논리 디스크, 메모리, USB 장치 | 기본 시스템 식별 |
| 계정 / 프로세스 / 서비스 | 활성 계정, 특정 프로세스, 서비스 상태 | 의심 프로세스 및 계정 선별 |
| 네트워크 / 설치 프로그램 | 네트워크 설정, IP 정보, 설치된 프로그램 | 침해 흔적과 설치 이력 확인 |
| 조건 기반 조회 | 속성 조합, 조건문, 반복 조회 | 정형화된 정보 수집 자동화 |

## 1.4 Sysinternals Suite
### 1.4.1 GUI vs 명령어 기반 도구
| 구분 | 장점 | 적합한 상황 |
| --- | --- | --- |
| GUI 기반 도구 | 직관적 시각화, 빠른 상태 파악 | 단일 시스템의 실시간 상태 확인 |
| 명령어 기반 도구 | 세부 옵션, 스크립트 자동화, 반복 수집 | 대량 시스템 분석 또는 정형화된 수집 절차 |

| 범주 | 도구 | 기능 |

# 2. 증거물을 이용한 실전 분석
## 2.1 Volatility 기반 메모리 포렌식
### 2.1.1 분석 목적
- 메모리 덤프와 Volatility 프레임워크를 활용해 레지스트리 정보를 추출한다.
- 사용자 계정, 세션 환경, 시스템 설정 관련 키를 확인한다.
- 침해 사고 분석에서 사용자 행위와 시스템 변경 흔적을 빠르게 확인한다.

### 2.1.2 명령어 구조
```bash
volatility -f [덤프파일] --profile=[OS버전] [플러그인] -K [레지스트리키]
```

| 옵션 / 인자 | 의미 |
| --- | --- |
| `-f [덤프파일]` | 분석할 메모리 덤프 파일 지정 |
| `--profile=[OS버전]` | 분석 대상 운영체제 버전 지정 |
| `[플러그인]` | 실행할 Volatility 플러그인 지정 |
| `-K [레지스트리키]` | 추출하거나 분석할 레지스트리 키 지정 |

### 2.1.3 활용 예시
| 항목 | 확인 내용 |
| --- | --- |
| 레지스트리 키 추출 | 메모리 덤프에서 레지스트리 키 추출 |
| Volatile Environment | 현재 로그인 사용자 환경 정보와 세션 환경 변수 |
| Profile List | 사용자 계정 정보와 사용자 프로파일 목록 |
| SID 분석 | 사용자 계정 식별, 계정 상태 확인, 레지스트리 경로 연결 |
| 덤프 파일 예시 | `win7.mem` |

### 2.1.4 주요 레지스트리 경로
| 경로 | 분석 목적 |
| --- | --- |
| HKCU / HKU | 현재 사용자와 전체 사용자 프로파일을 연결해 계정별 환경 정보 확인 |
| HKLM | `SAM`, `SECURITY`, `SOFTWARE`, `SYSTEM`, `BCD` 등 시스템 설정과 계정 관련 정보 확인 |
| HKCC | 부팅 시점에 동적으로 구성되는 현재 하드웨어 프로파일 정보 확인 |
