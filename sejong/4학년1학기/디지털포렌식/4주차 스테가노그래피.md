# 1. 스테가노그래피 개요
## 1.1 스테가노그래피란?
- **정의**: '감춰져 있는 쓰기'라는 의미를 가진다 (Stegano = 감춰져 있다, Graphy = 쓰기).
- 정보가 숨겨져 있는 상태로 상대방에게 전달하는 정보 은닉 기술이다.
## 1.2 스테가노그래피 구성요소
### 1.2.1 Carrier (캐리어)
- 데이터를 숨길 수 있는 매개체 역할을 하는 데이터 파일이다.
- **예시**: 이미지, 음악 파일, 문서 파일, HTML 등
- 마치 여행 가방(Carrier)에 짐을 넣듯이, 숨길 데이터를 담는 그릇의 역할을 한다.
### 1.2.2 Payload (페이로드)
- Carrier 내부에 실제로 숨기는 비밀 데이터이다.
- **예시**: 'sejong'과 같은 특정 텍스트, 혹은 숨겨진 비트 패턴(0101 등)
- 캐리어 내부에 넣는 "짐"의 역할을 수행한다.
## 1.3 크립토그래피 vs 스테가노그래피
### 1.3.1 크립토그래피 (Cryptography)
- 암호화 알고리즘을 이용하여 데이터를 알아볼 수 없게 숨겨서 주고받는 암호학 기술이다.
- 누구나 데이터가 암호화되어 '숨겨져 있다'는 사실 자체는 확인할 수 있다.
### 1.3.2 스테가노그래피 (Steganography)
- 데이터가 '존재한다'는 사실 자체를 숨기는 기술이다.
- **예시**: 텍스트의 띄어쓰기 규칙(1칸=0, 2칸=1)을 이용해 정보를 숨길 경우, 일반인에게는 평범한 텍스트로 보인다.
- 제3자는 숨겨진 데이터가 있다는 것조차 눈치채지 못하게 하는 것이 핵심이다.
## 1.4 스테가노그래피 사례
### 1.4.1 히스타이오스의 노예 문신
- 노예의 머리를 깎고 두피에 문신으로 메시지를 새긴 뒤, 머리카락이 자란 후 목적지로 보내 메시지를 은닉했다.
### 1.4.2 데마라투스 왁스판
- 왁스 아래에 있는 나무판에 글을 숨겨서 전달했다.
### 1.4.3 눈 깜빡임 (모스부호)
- 포로가 방송 중 짧고 긴 눈 깜빡임을 이용하여 'TORTURE(고문)'라는 메시지를 은밀히 전달했다.
### 1.4.4 악성코드 은닉 (2025년 최신 동향)
- 이미지나 멀티미디어 파일 내부에 악성 코드를 숨겨 보안 탐지를 우회한다. 겉보기에는 평범해 보이지만, 숨겨진 정보를 아는 공격자만 이를 해독하고 실행할 수 있다.
## 1.5 음파 기반 및 전파 기반 스테가노그래피
시각적 매체(이미지, 텍스트)뿐만 아니라 청각(음파), 전파 등 다양한 매체로 은닉 기술이 확장되고 있다.
### 1.5.1 음파 기반 스테가노그래피
- 음악이나 음성의 미세한 음의 높낮이, 음의 세기, 시간 간격의 변화를 주어 데이터를 은닉한다.
- **사례**: 인터넷 전화(VoIP)의 패킷 지연, 고/중저음 대역의 변조 등.
- FBI 등 정보 기관에서도 음성 파일 내 데이터 은닉 가능성을 경고하며 이를 잡아내기 위한 탐지 도구를 개발하고 있다.
### 1.5.2 전파 기반 스테가노그래피
- 인간은 느끼지 못하지만, 전파 신호의 미세한 조정을 통해 데이터를 은닉할 수 있다.
- **사례**:
    - **SIGSALY (2차 세계대전)**: 무작위 잡음을 삽입하여 적군이 도청해도 알 수 없는 비밀 음성 통신을 수행했다.
    - **러시아 스파이 사건**: 웹사이트에 올라온 평범한 이미지나 오디오 파일의 전파/주파수적 특징에 메시지를 은닉하여 지령을 전달했다.
# 2. 스테가노그래피 원리
## 2.1 이미지 원리
### 2.1.1 RGB 삼원색 구조
- 디지털 이미지는 R(Red), G(Green), B(Blue) 세 가지 색상 값의 조합으로 표현된다.
- 각 색상은 0~255(8bit)의 값을 가지며, 따라서 한 픽셀은 총 24bit (R8 + G8 + B8)로 구성된다.
- 사람의 눈은 1~2 정도의 아주 미세한 색상 값 변화를 거의 구별하지 못한다는 맹점을 이용한다.
### 2.1.2 데이터 은닉 방식
- 픽셀을 구성하는 RGB 값 중 가장 변화가 덜 티 나는 **최하위 비트**(LSB, Least Significant Bit)를 0 또는 1로 조작하여 비밀 데이터를 삽입한다.
- 하나의 이미지가 수천에서 수백만 개의 픽셀로 구성되어 있기 때문에, 방대한 비트(0101...) 데이터를 자연스럽게 분산하여 숨길 수 있다.
- 색상 자체는 육안으로 구별할 수 없을 만큼 거의 변하지 않지만, 숨겨진 데이터는 0과 1로 정확하게 저장된다.
## 2.2 문자 원리
### 2.2.1 문자를 이미지에 은닉
- 은닉할 텍스트(문자)는 먼저 아스키 코드(숫자)로 변환된다.
- 변환된 숫자는 다시 이진수(Binary)로 표현된다. (예: 문자 'H' = 10진수 72 = 2진수 01001000)
- 이렇게 변환된 이진수 형태의 텍스트 데이터를 이미지의 RGB 픽셀 하위 비트에 순차적으로 삽입한다.
## 2.3 스테가노그래피 코드 분석
### 2.3.1 코드 구조 이해
- **이중 for문**: 이미지의 높이(Height)와 너비(Width)를 기준으로 모든 픽셀을 순회한다.
- 각 픽셀의 R, G, B 값을 가져온 뒤, 산술 연산을 통해 각 색상 값의 최하위 비트(LSB)를 0으로 초기화하여 새로운 데이터를 넣을 '깨끗한 도화지' 상태를 만든다.
### 2.3.2 데이터 삽입 원리
- ASCII 문자를 이진수로 쪼갠 뒤, 준비된 RGB의 LSB 공간에 1비트씩 삽입한다.
- `switch`문 등을 사용하여 하나의 픽셀 안에서 R, G, B 채널 순서대로 비트를 차례차례 집어넣는다.
### 2.3.3 종료 조건
- 텍스트 문자열의 길이만큼 반복하며, 숨기고자 하는 모든 문자의 비트 삽입이 완료되면 특정한 종료 패턴(예: Null 비트 삽입)을 넣은 뒤 반복문을 종료한다.
### 2.3.4 데이터 추출 원리
- 데이터 삽입의 역순으로 진행된다. 이미지 전체를 순회하며 RGB의 LSB에서 1비트씩 데이터를 추출한다.
- 추출된 비트들을 8개씩 모아 조합하여 원래의 ASCII 문자로 재구성한다.
- 추출 과정에서 비트의 순서가 역방향이 되므로, `reverseBits`와 같은 함수를 사용하여 비트의 순서를 올바르게 뒤집는 처리를 수행한다.
## 2.4 그 외 스테가노그래피 기법
### 2.4.1 LSB 기법의 한계
- 단순한 LSB 기법은 비트를 순차적으로 삽입하므로 통계적인 분석(Steganalysis)을 통해 탐지될 확률이 높다.
- 데이터를 과도하게 넣을 경우 주변 픽셀과의 색상 차이가 커져 육안으로 발각될 위험이 존재한다.
### 2.4.2 고급 기법의 등장
- LSB의 단점을 보완하고 통계적 탐지를 회피하기 위해 더욱 복잡하고 정교한 기법들이 지속적으로 개발되고 있다.
- **대표 기법**: HUGO, WOW, S-UNIWARD 등
- 이러한 기법들은 단순한 순차적 대입이 아닌, 이미지의 복잡한 텍스처 영역 등 시각적/통계적 변화가 가장 적게 일어나는 최적의 픽셀을 수학적으로 계산하여 데이터를 숨기는 고도화된 알고리즘을 사용한다.
# 3. 스테가노그래피 실습
## 3.1 OpenStego 설치
### 3.1.1 OpenStego 개요
- 스테가노그래피 원리를 실습할 수 있도록 만들어진 대표적인 오픈 소스 도구이다.
- 직관적인 GUI(그래픽 사용자 인터페이스)를 제공하며, 소스 코드가 공개되어 있다.
- 사용자가 직접 코드를 분석하거나 수정하여 기능을 확장할 수 있다.
### 3.1.2 설치 방법
- 공식 Github 배포 페이지(https://www.google.com/search?q=https://github.com/syvaidya/openstego/releases)%EC%97%90%EC%84%9C 다운로드할 수 있다.
- Java 기반으로 작성되었으므로 실행을 위해 사전에 Java JDK 설치가 필수적이다.
## 3.2 OpenStego 실습
### 3.2.1 OpenStego 실행 화면 이해
- 메인 화면은 크게 **데이터 숨기기(Hide Data)**와 **데이터 추출하기(Extract Data)** 두 가지 핵심 기능으로 나뉜다.
- 단순한 텍스트뿐만 아니라 다른 문서, 이미지 등 '파일 단위'로도 은닉이 가능하다.
- 패스워드 설정 기능을 지원하여, 스테가노그래피(존재 은닉)와 크립토그래피(내용 암호화)를 강력하게 결합할 수 있다.
### 3.2.2 메시지 숨기기 (Hide data 설정)
- **준비물**: 데이터를 숨길 캐리어 역할의 '이미지 파일'과, 내부에 몰래 숨길 페이로드 역할의 '비밀 파일(텍스트 등)'을 준비한다.
- OpenStego의 `Hide Data` 메뉴에서 준비한 두 파일을 각각 지정한다.
- 보안을 높이기 위해 비밀번호를 설정할 수 있다(선택 사항).
- `Generate(생성)` 버튼을 클릭하면 원본 이미지와 육안으로는 거의 구별이 불가능한 스테가노그래피 적용 이미지가 새롭게 생성된다.
### 3.2.3 데이터 추출 (Extract data 설정)
- 데이터를 복원할 때는 `Extract Data` 메뉴를 사용한다.
- 앞서 생성된 스테가노그래피 이미지를 선택하고, 삽입 시 설정했던 패스워드를 정확히 입력한다.
- 정상적으로 인증되면 내부에 숨겨져 있던 파일이 손상 없이 그대로 복원되어 추출된다.
## 3.3 StegAnalysis (스테가노분석)
### 3.3.1 StegAnalysis란?
- 스테가노그래피 기술을 이용해 교묘하게 숨겨진 데이터를 찾아내고 식별하는 기술이다.
- 디지털 포렌식 관점에서 보았을 때 정보를 은닉하는 자를 쫓는 '추적자'의 역할을 담당한다.
- 겉보기엔 정상적인 파일들 사이에서, 통계적 이상점이나 구조적 변조를 파악하여 숨겨진 데이터의 존재 여부를 탐지해낸다.
### 3.3.2 StegAnalysis 도구
- 대표적인 분석 도구로 **Aletheia**, **VSL** 등이 있다.
- 학계 및 실무에서 사용하는 다양한 오픈소스 및 상용 도구가 존재한다.
- 대량의 이미지 파일 중에서 의심스러운 파일을 빠르게 걸러내는 '탐지 속도'와, 오탐 없이 정확하게 찾아내는 '탐지 정확도'가 이 도구들의 핵심 목표이다.
