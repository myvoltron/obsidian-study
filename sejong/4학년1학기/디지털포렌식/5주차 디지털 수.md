# 1. 디지털 데이터 해석의 필요성과 파싱 (1차시)
## 1.1 디지털 데이터와 진법
- **기본 원리**: 디지털 컴퓨터는 0과 1(2진법) 두 가지 상태로만 모든 정보를 저장, 전송, 처리합니다. 하지만 인간은 10진법을 사용하므로 진법 간의 변환과 해석이 반드시 필요합니다.
- **진법 묶음의 필요성**: 순수 2진 데이터(0101...)는 사람이 읽기 복잡하고 길기 때문에 여러 비트를 묶어 표현합니다.
    - 2비트 묶음 → 4진법
    - 3비트 묶음 → 8진법
    - **4비트 묶음 → 16진법 (가장 많이 사용)**
- **16진법의 특징 및 활용**:
    - 4bit를 1개의 16진 문자(0~F)로 표현하여 16개의 문자로 데이터를 나타냅니다.
    - 4개의 비트 패턴을 하나의 문자로 1:1 매핑할 수 있어, 이진 데이터를 컴팩트하고 쉽게 해석할 수 있습니다.
    - **예시**: `A472`와 같은 16진수 데이터는 실제로는 2진수로 저장되어 있으며, 규격(예: DOS Time)에 따라 생성/접근/수정 시간 등의 정보로 해석됩니다.
## 1.2 데이터 해석 및 파싱의 중요성
디지털 포렌식은 디지털 데이터를 분석해 사건을 규명하는 분야이므로, 2진 데이터를 정확히 해석하는 것이 핵심 기술입니다.
- **해석 방식에 따른 차이**: 메모리에 저장된 동일한 2진수 값이라도 시간, 정수, 소수 등 어떤 방식으로 해석하느냐에 따라 전혀 다른 의미를 갖게 됩니다.
- **파싱(Parsing)의 역할**: 데이터가 저장된 기법을 파악하고, 특정 규칙에 따라 2진 데이터를 사람이 이해할 수 있는 정보로 변환하는 과정입니다. (예: 시스템의 2진 데이터를 DOS Time 규칙에 맞춰 `연도/월/일/시/분/초`로 분리해 해석)
- **자동화의 필요성**: 방대한 데이터를 수작업으로 해석하면 시간이 과다하게 소모되므로, 정확한 해석 규칙을 기반으로 소프트웨어 도구를 구현하여 자동 파싱을 수행해야 합니다.
## 1.3 디지털 데이터의 단위
- **물리적 단위 (저장 관점)**:
    - **Bit(비트)**: 0 또는 1을 나타내는 최소 단위.
    - **Byte(바이트)**: 8bit = 1Byte. 가장 널리 쓰이는 기본 저장 단위입니다.
    - **용량의 증가**: $2^{10} (1024)$을 기준으로 증가합니다. ($KB \rightarrow MB \rightarrow GB \rightarrow TB \rightarrow PB \rightarrow ZB$)
    - **Word(워드)**: CPU가 한 번에 처리할 수 있는 데이터의 크기를 의미합니다. CPU 발전에 따라 4bit → 8bit → 16bit → 32bit → 64bit로 발전해 왔습니다. (Double Word 등은 처리 단위의 확장을 의미)
- **논리적 단위 (의미 중심)**:
    - 파일, 문서, 데이터 구조 등 의미를 중심으로 묶인 단위입니다.
    - 예: 데이터베이스나 스프레드시트의 필드(Field)와 레코드(Record).
# 2. 디지털 데이터 수 체계와 엔디언 (2차시)
## 2.1 디지털 데이터 수 체계
- **2진수** $\rightarrow$ **10진수 해석 원리**: 각 비트의 위치는 2의 거듭제곱($2^n$) 자리값을 가집니다. ($2^0=1, 2^1=2, 2^2=4, 2^3=8...$). 값이 '1'인 자리의 값만 모두 더하면 10진수로 변환됩니다.
- **표현 범위와 확장**: 1Byte(8bit)가 표현할 수 있는 범위는 `00000000` ~ `11111111` (10진수 0~255)입니다. 최대값의 한계가 존재하므로 더 큰 숫자를 저장하기 위해 2Byte, 4Byte 단위로 확장하여 사용합니다.
- **저장 방식**: 메모리는 항상 Byte 단위로 쪼개진 공간이며, 큰 데이터는 바이트 단위로 나뉘어 저장됩니다. 따라서 해석 시 각 바이트를 올바르게 조합해야 원래 값을 파악할 수 있습니다.
## 2.2 바이트 저장 순서: 엔디언 (Endian)
CPU의 설계 종류에 따라 데이터를 메모리에 저장하는 순서(엔디언)가 다르기 때문에 포렌식 분석 시 반드시 확인해야 합니다.
- **Big-Endian (빅 엔디언)**:
    - 바이트열에서 가장 큰 자릿값을 먼저 저장하는 방식입니다. (메모리 하위 주소에 큰 바이트 저장)
    - 사람이 숫자를 읽는 방향과 같아 가독성이 뛰어납니다.
- **Little-Endian (리틀 엔디언)**:
    - 바이트열에서 가장 작은 자릿값을 먼저 저장하는 방식입니다. (메모리 하위 주소에 작은 바이트 저장)
    - 인텔(Intel), AMD 등 대부분의 대중적인 CPU에서 사용합니다.
    - 데이터 형 변환이나 크기 확장(예: 2바이트 $\rightarrow$ 4바이트) 시 구조 변경이 쉽다는 수학적 장점이 있습니다.
- **포렌식 시 주의점**: 동일한 바이트 배열이라도 Endian 방식에 따라 완전히 다른 숫자로 해석됩니다. 따라서 원시 데이터를 분석할 때는 반드시 해당 시스템(CPU)의 Endian 방식을 파악해야 합니다.
# 3. 진법 변환과 음수/소수점 표현 (3차시)
## 3.1 16진수 (Hexadecimal)
- **Raw Data 표현의 최적화**: 2진수는 너무 길어 사람이 식별하기 어렵습니다. 디지털 시스템의 구조(2, 4, 8, 16)와 완벽히 맞아떨어지는 16진수가 원시 데이터 분석에 가장 적합합니다.
- **변환의 용이성**: 2진수를 10진수로 바꿀 때는 자리값 곱셈과 덧셈이 필요해 복잡하지만, 2진수 $\rightarrow$ 16진수 변환은 4비트씩 끊어서 표와 1:1 매핑만 하면 되므로 매우 직관적입니다. (예: `0101` $\rightarrow$ `5`, `1110` $\rightarrow$ `E`, `1010` $\rightarrow$ `A`, `0001` $\rightarrow$ `1` $\Rightarrow$ `5EA1`)
- 컴퓨터와 포렌식 분석 도구들은 이러한 이유로 16진수 기반의 데이터 표시를 선호합니다.
## 3.2 음수 표현 방법
컴퓨터는 물리적으로 0과 1만 저장할 수 있으므로, 뺄셈 연산과 음수를 표현하기 위해 특별한 수학적 규칙을 도입했습니다.
1. **부호-크기 (Sign-Magnitude)**:
    - 가장 왼쪽 1비트를 부호비트($0=+, 1=-$)로 사용하고, 나머지 7비트를 절댓값으로 사용합니다.
    - **단점**: 직관적이지만 숫자 표현 범위가 줄어들고, $+0$(`00000000`)과 $-0$(`10000000`) 두 가지의 0이 중복해서 존재하는 치명적인 결함이 있어 실제 시스템에서는 사용되지 않습니다.
2. **1의 보수 (1's Complement)**:
    - 모든 비트를 반전시키는 방식 ($0 \rightarrow 1, 1 \rightarrow 0$).
    - 예: $+11$(`00001011`) $\rightarrow$ 반전 $\rightarrow$ `11110100`($-11$).
    - **단점**: 여전히 $+0$과 $-0$ 두 개가 존재합니다. 2의 보수를 만들기 위한 중간 과정의 개념입니다.
3. **2의 보수 (2's Complement) - ★실제 컴퓨터 사용 방식**:
    - **규칙**: 1의 보수(비트 반전) 연산 후 마지막에 `+1`을 더합니다. (발생한 가장 높은 자리의 올림수는 버림)
    - **장점**:
        - $0$이 단 하나만 존재합니다.
        - 뺄셈을 덧셈 연산으로 처리할 수 있습니다. (예: $5-3$ 을 $5+(-3)$으로 계산)
        - 이로 인해 덧셈기(Adder) 회로 하나만으로 덧셈과 뺄셈 등 모든 산술 연산을 구현할 수 있어 매우 효율적입니다.
## 3.3 소수점 표현 방법
1. **고정 소수점 (Fixed Point)**:
    - 숫자의 비트를 `부호(Sign) + 정수부 + 소수부`로 나누어 고정된 위치에 소수점이 있다고 가정하는 방식입니다.
    - 직관적이고 이해하기 쉽지만, 표현할 수 있는 수의 범위가 제한적이어서 현대 컴퓨터 시스템에서는 교육용 외에는 거의 사용되지 않습니다.
2. **부동 소수점 (Floating Point)**:
    - 소수점의 위치를 고정하지 않고 '떠다니게(Floating)' 하여, 아주 큰 숫자나 아주 미세하게 작은 숫자를 효율적인 메모리 공간 안에 표현하는 방식입니다. (IEEE 754 표준 등)
    - **구조**: `부호(Sign) + 지수(Exponent) + 가수(Mantissa)`
    - **표현 원리**: 숫자를 $1.76 \times 10^5$와 같은 과학적 표기법으로 정규화합니다.
    - **규칙**: 정규화된 2진수는 항상 `1.XXXX` 형태로 시작한다는 점을 이용해, 맨 앞의 `1`은 메모리에 저장하지 않고 생략(Hidden bit)하여 가수(Mantissa) 부분의 공간을 절약합니다.
    - **장점**: 작은 메모리 용량으로 엄청나게 넓은 범위의 수를 표현할 수 있어 연산 효율이 좋고 범용적으로 사용됩니다.
