# 1. 디지털 수/문자
## 1.1 디지털 데이터와 진법
### 1.1.1 디지털 데이터의 기본
- 디지털 시스템은 모든 정보를 0과 1로 저장한다.
- 사람이 원시 데이터를 해석하려면 2진 데이터를 8진수, 10진수, 16진수 등으로 변환할 수 있어야 한다.

### 1.1.2 진법 요약
| 구분 | 묶음 기준 | 특징 |
| --- | --- | --- |
| 2진법 | 1bit | 컴퓨터 내부 저장 단위 |
| 8진법 | 3bit | 2진 데이터를 짧게 표현 |
| 10진법 | 사람의 일반 계산 체계 | 사람이 읽기 쉬움 |
| 16진법 | 4bit | Raw Data와 1:1 매핑이 쉬워 포렌식에서 자주 사용 |

### 1.1.3 16진수 활용
- 16진수 1자리는 2진수 4bit와 대응된다.
- 예를 들어 `0101 1110 1010 0001`은 `5EA1`로 표현할 수 있다.
- 헥스 뷰어, 파일 시그니처, 시간 값 해석 등 원시 데이터 분석에서 기본 표현 방식으로 쓰인다.

## 1.2 데이터 해석과 파싱
### 1.2.1 해석 방식의 중요성
- 같은 비트열도 정수, 문자, 시간, 소수 등 해석 기준에 따라 전혀 다른 의미가 된다.
- 포렌식 분석에서는 데이터가 어떤 구조와 규칙으로 저장되었는지 먼저 확인해야 한다.

### 1.2.2 파싱의 역할
| 항목 | 내용 |
| --- | --- |
| 목적 | 2진 데이터를 사람이 이해할 수 있는 값으로 변환 |
| 기준 | 파일 포맷, 시간 포맷, 인코딩 방식, 데이터 구조 |
| 예시 | DOS Time 값을 연·월·일·시·분·초로 분리 |
| 실무 의미 | 반복 해석을 자동화하여 대량 증거 분석 시간을 줄임 |

## 1.3 디지털 데이터 단위
### 1.3.1 기본 단위
| 단위 | 의미 |
| --- | --- |
| bit | 0 또는 1 하나를 저장하는 최소 단위 |
| Byte | 8bit, 데이터 저장의 기본 단위 |
| Word | CPU가 한 번에 처리하는 데이터 크기 |
| KB/MB/GB/TB | 1024배 단위로 증가하는 저장 용량 단위 |

### 1.3.2 물리적 단위와 논리적 단위
| 구분 | 설명 | 예시 |
| --- | --- | --- |
| 물리적 단위 | 하드웨어 저장 관점의 단위 | bit, Byte, sector |
| 논리적 단위 | 사용자가 의미를 부여한 데이터 묶음 | 파일, 문서, 레코드 |

# 2. 디지털 수 해석
## 2.1 수 체계와 바이트 저장
### 2.1.1 표현 범위
- 1Byte는 8bit이므로 `00000000`부터 `11111111`까지 표현한다.
- 부호 없는 정수 기준 표현 범위는 0~255이다.
- 더 큰 값을 저장하려면 2Byte, 4Byte, 8Byte처럼 저장 공간을 확장한다.

### 2.1.2 2진수 해석
- 각 비트 위치는 $2^n$의 자리값을 가진다.
- 값이 1인 자리의 값만 더하면 10진수로 변환된다.

## 2.2 Big-Endian & Little-Endian
### 2.2.1 Endian 비교
| 구분 | 저장 방식 | 특징 | 포렌식 주의점 |
| --- | --- | --- | --- |
| Big-Endian | 큰 자릿값 바이트를 먼저 저장 | 사람이 읽는 숫자 순서와 유사 | 네트워크 프로토콜 등에서 자주 사용 |
| Little-Endian | 작은 자릿값 바이트를 먼저 저장 | 형 변환과 산술 연산에 유리 | Intel, AMD 계열 시스템 해석 시 중요 |

### 2.2.2 해석 시 주의점
- 같은 헥스 값도 Endian 방식에 따라 전혀 다른 숫자가 된다.
- Raw Data를 읽을 때는 데이터를 기록한 시스템의 CPU 구조와 파일 포맷 규칙을 함께 확인해야 한다.

# 3. 디지털 수 파싱
## 3.1 음수 표현 방법
### 3.1.1 음수 표현 비교
| 방식 | 원리 | 한계 / 특징 |
| --- | --- | --- |
| 부호-크기 | 최상위 비트를 부호로 사용 | `+0`과 `-0`이 따로 존재 |
| 1의 보수 | 양수 비트를 모두 반전 | 여전히 0이 두 개 존재 |
| 2의 보수 | 비트 반전 후 1을 더함 | 현대 컴퓨터의 표준 방식 |

### 3.1.2 2의 보수의 의미
- 0을 하나만 표현할 수 있고, 뺄셈을 덧셈 회로로 처리할 수 있다.
- 하드웨어 구현이 단순해 현재 컴퓨터 산술 연산의 기본 방식으로 사용된다.

## 3.2 소수점 표현 방법
### 3.2.1 고정 소수점과 부동 소수점
| 구분 | 저장 방식 | 특징 |
| --- | --- | --- |
| 고정 소수점 | 부호, 정수부, 소수부 위치를 고정 | 이해하기 쉽지만 표현 범위가 좁음 |
| 부동 소수점 | 부호, 지수, 가수로 분리 | 매우 큰 수와 작은 수를 효율적으로 표현 |

### 3.2.2 부동 소수점 핵심
- 숫자를 과학적 표기법처럼 정규화하여 저장한다.
- IEEE 754에서는 `부호(Sign) + 지수(Exponent) + 가수(Mantissa)` 구조를 사용한다.
- 정밀도 한계가 있으므로 분석 시 반올림 오차 가능성을 고려해야 한다.
