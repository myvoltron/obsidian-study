# 1. 스토리지와 데이터 획득 방법에 따른 디지털 포렌식 이슈
## 1.1 디스크 수집 및 증거물 종류
디지털 포렌식에서 데이터 획득은 분석 대상 장치, 저장매체 구조, 획득 도구, 획득 기법에 따라 결과가 달라질 수 있다. 따라서 증거 획득 전 대상 시스템의 구조와 저장매체 현황을 먼저 파악해야 한다.
### 1.1.1 디지털 증거 획득의 전체 흐름
- 다양한 컴퓨터 시스템에 스토리지가 존재한다.
    - PC
    - 서버
    - 스마트폰
    - IoT 장비
- 데이터 획득은 도구, 대상 시스템, 획득 기법에 따라 결과가 달라질 수 있다.
- 같은 증거물이라도 수집 방식이 다르면 해시값이나 수집 범위가 달라질 수 있다.
- 증거 획득 전 대상 시스템의 종류와 저장매체 구성을 확인해야 한다.
- 내부 저장장치와 외부 저장장치의 존재 여부를 확인해야 한다.
- 어떤 데이터를 어떤 방식으로 수집할지 결정해야 한다.
### 1.1.2 대상 시스템에 따른 주요 이슈
- **장치별 증거 보관 위치 파악**
    - 장치마다 증거가 저장되는 위치가 다를 수 있다.
    - PC, 서버, 스마트폰, IoT 장비는 저장 구조가 다르다.
    - 증거 보관 위치를 파악해야 누락 없이 수집할 수 있다.
- **스토리지 분리 확보 필요**
    - 내장 메모리를 확인해야 한다.
    - SD 카드를 확인해야 한다.
    - USIM을 확인해야 한다.
    - 스마트폰이나 임베디드 장비는 저장매체가 여러 개일 수 있다.
    - 저장매체별로 증거가 분산되어 있을 수 있다.
- **네트워크 연결 차단 필요**
    - 스마트폰 등 네트워크 연결 장치는 네트워크 차단이 필수이다.
    - 원격 삭제, 원격 잠금, 데이터 변경을 방지해야 한다.
    - 전자파 차단 포켓을 활용할 수 있다.
- **기타 장치의 내부 저장매체 분석**
    - 서버, 복합기, 프린터도 내부 저장매체 분석 대상이다.
    - 출력 기록, 스캔 기록, 내부 캐시, 저장 파일 등이 존재할 수 있다.
### 1.1.3 데이터 획득 기법 분류
데이터 획득 기법은 크게 물리적 획득과 논리적 획득으로 나눌 수 있다.
- **물리적 획득(Physical Acquisition)**
    - 저장장치 전체를 비트 단위로 추출하는 방식이다.
    - 운영체제 파일 시스템 관점이 아니라 저장매체 전체 관점에서 접근한다.
    - 삭제되었거나 파일 시스템에서 보이지 않는 데이터까지 확보할 수 있다.
- **CHIP-OFF**
    - 저장 칩을 장치에서 물리적으로 분리한 뒤 데이터를 추출하는 방식이다.
    - 침습적인 방식이다.
    - 장치가 정상적으로 동작하지 않아도 데이터를 추출할 가능성이 있다.
    - 파손, 침수, 고의 파괴, 일반적인 논리적 접근이 불가능한 상황에서 활용할 수 있다.
    - 절차는 기기 분해 -> 메모리 칩 추출 -> 칩 클리닝 및 마운트 -> 데이터 추출 순서이다.
- **JTAG**
    - 칩의 디버깅 포트를 이용하는 비침습적 방식이다.
    - 칩을 분리하지 않고 저수준 메모리 접근이 가능할 수 있다.
    - Pin Emulation을 이용해 데이터를 추출한다.
    - 절차는 JTAG 핀 식별 -> JTAG 리더기 연결 순서이다.
    - 일부 기기는 JTAG 포트가 숨겨져 있거나 비활성화되어 있을 수 있다.
- **논리적 획득(Logical Acquisition)**
    - 운영체제가 인식하는 영역을 중심으로 데이터를 추출하는 방식이다.
    - 파일 시스템 기반으로 데이터를 추출한다.
    - 사용자가 접근 가능한 파일과 시스템이 제공하는 데이터를 중심으로 수집한다.
    - 비교적 빠르게 수집할 수 있고 장치 손상 가능성이 낮다.
    - 삭제 데이터나 숨겨진 영역은 수집되지 않을 수 있다.
### 1.1.4 부트로더 및 플래싱 도구 활용
부트로더와 플래싱 도구는 스마트폰 등 임베디드 장치에서 데이터를 획득할 때 활용될 수 있다.
- **부트로더 단계 활용**
    - 보안 설정에 따라 파티션 접근이 가능할 수 있다.
    - 운영체제 부팅 이전 단계에서 특정 파티션을 읽을 수 있는 경우가 있다.
    - 장치의 잠금 상태와 보안 정책에 따라 접근 가능 범위가 달라질 수 있다.
    - 비침습적 방식이다.
- **플래싱 도구**
    - 플래시 메모리에 펌웨어 이미지나 파티션 이미지를 기록하기 위한 도구이다.
    - 펌웨어 또는 파티션 이미지를 읽는 데 활용될 수 있다.
    - 잠금 상태의 스마트폰에서도 일부 데이터 획득이 가능할 수 있다.
### 1.1.5 획득 시 고려 사항
디지털 증거 획득에서는 수집 도구, 수집 옵션, 대상 범위, 숨겨진 영역 포함 여부를 명확히 해야 한다.
- 도구별로 수집하는 영역이 다를 수 있다.
- 수집 영역 차이로 인해 해시값이 불일치할 수 있다.
- 동일한 저장장치를 대상으로 해도 도구와 옵션에 따라 결과가 다를 수 있다.
- 숨겨진 스토리지 영역이 수집 대상에 포함되는지 확인해야 한다.
- 원본성을 입증하기 위해 사용 도구, 사용 옵션, 수집 대상, 수집 범위, 수집 방식을 명시해야 한다.
- PC / 서버의 전체 디스크 이미징 절차는 3주차 실습 내용과 연결된다.
- 스마트폰은 내장 메모리, 외부 카드, USIM, 네트워크 차단 여부를 함께 고려한다.
- IoT / 복합기는 내부 저장매체와 장치 내부의 로그, 설정, 파일 데이터를 분석 대상으로 본다.
# 2. 검색/통계/시각화 기술
## 2.1 포렌식 검색 및 통계 기술
디지털 증거 분석에서 검색, 통계, 시각화는 핵심 기술이다. 검색과 통계는 수집된 데이터에서 의미 있는 정보를 정제하고 추출하는 역할을 하며, 시각화는 분석 결과를 구조화하고 이해 가능한 형태로 표현하는 역할을 한다.
### 2.1.1 디지털 증거 분석 기술의 필요성
- 디지털 증거 분석의 핵심 기술은 검색, 통계, 시각화이다.
- **검색·통계**
    - 수집된 데이터에서 의미 있는 정보를 정제한다.
    - 수집된 데이터에서 의미 있는 정보를 추출한다.
    - 사건과 관련 없는 데이터를 제외하는 데 활용된다.
    - 분석 범위를 줄이고 효율성을 높인다.
- **시각화**
    - 분석 결과를 구조화한다.
    - 분석 결과를 이해 가능한 형태로 표현한다.
    - 복잡한 데이터를 사람이 해석하기 쉽게 만든다.
- 스마트폰과 PC 저장장치가 대용량화되면서 전체 데이터를 전수조사하는 것은 비현실적이다.
- 사건 관련 정보만 빠르게 선별하고 불필요한 데이터를 배제하는 것이 중요하다.
### 2.1.2 대용량 저장매체 환경에서 검색 기술의 중요성
- 사건과 관련된 자료를 빠르게 식별한다.
- 불필요한 데이터를 제외한다.
- 분석 대상을 줄인다.
- 중요한 증거를 빠르게 찾는다.
- 저장매체가 대용량화되면서 전체 데이터 분석이 사실상 불가능하다.
- 검색 기술을 통해 분석 대상 범위를 축소하고 심도 있는 분석을 수행할 수 있다.
### 2.1.3 검색 기술의 세 가지 형태
포렌식 검색 기술은 일반 검색, 해시 검색, 슬랙 검색으로 구분할 수 있다.
- **일반 검색(키워드 기반)**
    - 입력 문자열, 파일명, 파일 속성, 파일 내부 문자열을 기반으로 데이터를 찾는다.
    - 다양한 인코딩과 바이너리 형태를 고려해야 한다.
    - Linux `grep`, OS indexing 기능 등이 예시이다.
- **해시 검색(Hashed Search)**
    - 알려진 파일의 해시값 데이터베이스와 비교해 파일을 식별하거나 불필요한 파일을 제외하는 방식이다.
    - 대량 데이터 필터링에 효율적이다.
    - 정상 파일이나 알려진 파일을 빠르게 제외해 분석자가 집중해야 할 파일 범위를 줄일 수 있다.
    - NSRL은 NIST 프로젝트이며, 알려진 실행 파일 해시값을 정리한 데이터베이스이다.
- **슬랙 검색(Slack Search)**
    - 파일 시스템의 잉여 공간인 slack space를 탐색하는 방식이다.
    - 삭제되었거나 일부만 남은 파일 흔적을 식별한다.
    - 파일 시스템에서 일반적으로 보이지 않는 잔존 데이터를 찾는다.
### 2.1.4 시각화 기술
시각화 기술은 분석 결과를 가시화하는 기술이다. 강의자료에서는 이를 데이터 `꽃 피우기`라고 표현한다.
- 분석 결과를 이해하기 쉬운 형태로 표현한다.
- 복잡한 분석 결과를 구조화한다.
- 파일 접근 시간과 수정 시간을 시간 순으로 재구성한다.
- 시스템 사용 패턴과 사용자 행위를 추론한다.
- 컴퓨터를 언제 켰는지, 주말 사용 여부, 특정 시간대 파일 접근 여부 등을 확인할 수 있다.
### 2.1.5 통계 분석 기술
통계 분석 기술은 많은 데이터의 패턴을 양적으로 파악해 고급 정보를 생성하는 기술이다.
- 단순 나열된 데이터를 집계한다.
- 의미 있는 경향성을 추출한다.
- 시간별, 요일별, 파일 종류별 집계에 활용한다.
- 타임라인 데이터 통계를 확장해 요일별 컴퓨터 사용량 등을 분석할 수 있다.
### 2.1.6 분석 기술 연계
검색, 통계, 시각화는 개별적으로도 활용되지만 전체 분석 과정에서는 순차적으로 연계된다.
```text
검색 -> 통계 -> 시각화
```
- **검색**: 사건 관련 데이터를 선별하고 불필요한 데이터를 제외한다.
- **통계**: 데이터를 수치화하고 시간, 요일, 파일 종류 등 기준별로 패턴을 분석한다.
- **시각화**: 분석 결과를 타임라인, 그래프, 차트 등 이해하기 쉬운 형태로 표현한다.
- 전체 분석 과정에서 효율적으로 증거를 확인하고, 수사나 보고 과정에서 결과 전달력을 높일 수 있다.
# 3. 시스템 로그와 안티 포렌식
## 3.1 시스템 로그
시스템 로그는 시스템과 사용자 행위를 지속적으로 기록한 데이터이다. 디지털 포렌식에서는 시스템에 남겨진 기록을 통해 행위를 추적하는 핵심 증거로 활용된다.
### 3.1.1 시스템 로그와 안티 포렌식 개요
- 디지털 포렌식은 `쫓는 자 vs 쫓기는 자`의 구조로 이해할 수 있다.
- 쫓는 자는 시스템에 남겨진 기록을 통해 행위를 추적한다.
- 쫓기는 자는 흔적을 지우기 위한 기술을 사용한다.
- 흔적을 지우기 위한 기술이 안티 포렌식이다.
- 시스템 로그는 디지털 블랙박스이다.
### 3.1.2 시스템 로그의 개념
- 시스템 및 사용자 행위를 지속적으로 기록한 데이터이다.
- 운영체제, 응용 프로그램, 보안 기능 등이 남기는 기록이다.
- 시간 순서에 따라 사건을 재구성하는 데 활용된다.
- 사용자 로그인 기록, 명령 실행 기록, 시스템 상태, 오류, 보안 이벤트를 확인할 수 있다.
- 침해사고 조사와 사용자 행위 추적의 핵심 증거이다.
### 3.1.3 리눅스 / 유닉스 계열 시스템 로그
리눅스와 유닉스 계열 시스템은 운영체제별로 로그 위치와 형식이 다를 수 있다. 주요 로그는 사용자 행위와 시스템 이벤트를 동시에 기록한다.
- **acct / pact**: 사용자 명령어 실행 기록을 저장하며, 어떤 명령이 실행되었는지 추적하는 데 활용된다.
- **lastlog**: 최근 로그인 기록을 저장하며, 사용자의 마지막 로그인 정보를 확인할 수 있다.
- **loginlog**: 로그인 실패 기록을 저장하며, 비정상 로그인 시도나 계정 공격 흔적을 확인하는 데 활용된다.
- **messages**: 시스템 메시지와 부팅 메시지를 저장하며, 시스템 상태나 오류를 확인하는 데 활용된다.
- **sulog**: `su` 명령어 사용 기록을 저장하며, 권한 전환 시도나 관리자 권한 사용 흔적을 확인하는 데 활용된다.
### 3.1.4 윈도우 시스템 로그
윈도우 시스템 로그는 이벤트 로그 형태로 관리된다. 로그 단위는 Event이며, 이벤트 뷰어를 통해 통합 관리된다.
- 이벤트 ID를 기준으로 필터링할 수 있다.
- 등급을 기준으로 필터링할 수 있다.
- **보안 로그**: 로그인 성공/실패, 파일 접근, 보안 관련 이벤트를 확인하는 데 사용된다.
- **시스템 로그**: 부팅 기록, 장치 연결 기록, 오류 기록을 저장하며 시스템 상태 분석에 활용된다.
- **응용 프로그램 로그**: 프로그램 오류, 실행 상태, 경고 등 응용 프로그램에서 발생한 이벤트를 저장한다.
### 3.1.5 웹 로그
웹 로그는 웹 브라우저 사용 흔적을 저장한다. 웹 아티팩트 분석을 통해 사용자 행위를 추적할 수 있다.
- 방문 기록, 캐시, 쿠키, 브라우저 사용 흔적을 확인할 수 있다.
- 사용자가 어떤 웹사이트를 방문했는지 확인할 수 있다.
- 어떤 파일이나 리소스가 캐시에 남았는지 확인할 수 있다.
- 쿠키를 통해 세션 유지나 접속 흔적을 분석할 수 있다.
- 웹 아티팩트 분석으로 사용자 행위 추적이 가능하다.
## 3.2 데이터 복구 회피 기술
데이터 복구 회피 기술은 포렌식 분석을 방해하거나 무력화하기 위한 기술이다. 안티 포렌식의 주요 영역 중 하나이다.
### 3.2.1 안티 포렌식 개념
- 포렌식 분석을 방해하거나 무력화하기 위한 기술이다.
- 증거가 발견되거나 복구되는 것을 어렵게 만든다.
- 목적은 증거 삭제, 증거 은닉, 증거 훼손이다.
- 공격자가 활용할 수 있으며, 보안 목적이나 정보 보호 목적으로도 활용될 수 있다.
### 3.2.2 주요 안티 포렌식 기법
- **데이터 영구 삭제**
    - 데이터를 복구할 수 없도록 삭제한다.
    - Disk Wiping 방식을 사용할 수 있다.
    - Degausser를 활용해 강력한 자기장으로 자기 저장매체의 데이터를 파괴할 수 있다.
- **자동 삭제 프로그램**
    - 복구 불가능하도록 데이터를 덮어쓴다.
    - 특정 조건이나 주기에 따라 데이터를 자동 삭제할 수 있다.
- **암호화**
    - 파일이나 압축 파일을 암호화한다.
    - 키 없이는 복호화가 어려워 분석자가 내용을 확인하기 어렵다.
- **스테가노그래피**
    - 이미지 등에 데이터를 은닉한다.
    - 겉으로 보기에는 일반 파일처럼 보이게 만들어 데이터 존재 자체를 숨긴다.
### 3.2.3 디스크 덮어쓰기 기법
디스크 덮어쓰기는 기존 데이터를 다른 값으로 반복 기록해 물리적 잔존 데이터를 제거하는 방식이다.
- **단순 0 덮어쓰기**
    - 디스크 영역을 0으로 덮어쓴다.
    - 일부 복구 가능성이 존재할 수 있다.
    - 단일 패턴 덮어쓰기는 복구 회피 측면에서 한계가 있을 수 있다.
- **다중 덮어쓰기**
    - 서로 다른 패턴으로 반복 기록한다.
    - 복구 가능성을 낮춘다.
    - 단순 0 덮어쓰기의 한계를 보완한다.
- **목적**
    - 물리적 잔존 데이터를 제거한다.
    - 삭제된 데이터의 복구 가능성을 낮춘다.
    - 포렌식 분석을 어렵게 만든다.
