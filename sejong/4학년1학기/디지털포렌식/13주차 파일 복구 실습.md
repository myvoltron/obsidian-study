# 1. FAT 파일 삭제
## 1.1 FAT 파일 삭제 동작 원리
### 1.1.1 삭제 시 변경되는 영역
| 영역 | 변경 내용 | 포렌식 의미 |
| --- | --- | --- |
| FAT 엔트리 | 파일이 사용하던 클러스터 엔트리를 `0x00`으로 초기화 | 클러스터 연결 정보가 사라짐 |
| Directory Entry | 파일 이름 첫 글자를 `0xE5`로 변경 | 삭제 파일 식별 단서 |
| Data Area | 파일 데이터 자체는 그대로 유지 | 덮어쓰기 전까지 복구 가능성 존재 |

### 1.1.2 삭제 연산의 특징
- FAT 파일시스템의 삭제 연산은 실제 데이터를 즉시 지우지 않는다.
- 예시 클러스터가 190번, 191번이라면 해당 FAT 엔트리만 초기화된다.
- Data Area에 저장된 파일 데이터가 남아 있기 때문에 삭제 직후 복구 가능성이 존재한다.

# 2. 삭제된 FAT 파일 복구
## 2.1 복구 가능 조건
### 2.1.1 복구 단서
| 단서 | 유지 여부 | 복구에 주는 영향 |
| --- | --- | --- |
| Data Area | 유지될 수 있음 | 실제 파일 내용을 읽을 수 있다. |
| Directory Entry | 첫 글자만 `E5`로 변경 | 파일 크기, 시간, 속성, 시작 클러스터가 남을 수 있다. |
| FAT 엔트리 | `0x00`으로 초기화 | 연결 정보는 사라지지만 연속 저장 파일은 복구 가능하다. |

### 2.1.2 삭제 파일 식별
- 포렌식 도구에서는 `E5` 값을 기준으로 삭제된 파일을 식별한다.
- FAT 엔트리가 초기화되었더라도 파일이 연속된 클러스터에 저장되어 있었다면 시작 클러스터와 파일 크기 정보로 복구할 수 있다.

## 2.2 복구 시나리오
### 2.2.1 메타데이터가 모두 있는 경우
| 단계 | 작업 | 결과 |
| --- | --- | --- |
| 1 | Directory Entry에서 `E5` 삭제 엔트리 탐색 | 삭제 파일 후보 식별 |
| 2 | 파일 시작 클러스터, 파일 크기, 속성, 시간 정보 확인 | 복구에 필요한 메타데이터 확보 |
| 3 | Data Area에서 시작 클러스터부터 파일 크기만큼 읽기 | 원본 파일 데이터 획득 |
| 4 | 읽은 데이터를 파일로 저장 | 빠르고 정확한 복구 가능 |

### 2.2.2 메타데이터가 없는 경우
| 조건 | 복구 방식 | 한계 |
| --- | --- | --- |
| Directory Entry 정보 없음 | Data Area의 클러스터 조각 분석 | 복구 시간 증가 |
| FAT 정보 없음 | 파일 시그니처 기반으로 다음 클러스터 추정 | 정확도 저하 |
| 파일이 조각남 | 퍼즐 맞추기 방식의 재구성 필요 | 복구 난이도 급상승 |

### 2.2.3 연속 클러스터 여부
| 저장 상태 | 복구 난이도 | 이유 |
| --- | --- | --- |
| 연속 저장 | 낮음 | 시작 클러스터와 크기 정보만으로 복구 가능 |
| 조각난 저장 | 높음 | 다음 클러스터 추정과 결과 검증이 필요 |

### 2.2.4 삭제 후 덮어쓰기 발생 시
- 삭제된 클러스터 영역인 Data Area에 새로운 파일이 기록될 수 있다.
- 새로운 파일이 기록되면 기존 데이터가 덮어써진다.
- 원본 데이터가 소실되면 복구가 불가능하다.
