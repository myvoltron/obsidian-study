# 1차시: 윈도우 시스템 부팅 프로세스
## 1. 부팅 프로세스
윈도우 시스템의 부팅은 전원이 인가된 이후 펌웨어, 부트 코드, 운영체제 이미지, 커널, 초기화 프로세스, 로그인 화면으로 이어지는 단계적 흐름이다.
### 1.1 부팅 단계
1. **전원 인가**
    - 컴퓨터에 전원이 들어온다.
    - CPU가 0번지 데이터를 읽는다.
    - BIOS 실행을 준비하는 단계이다.
2. **BIOS 실행**
    - 하드웨어 점검을 수행한다.
    - CPU, 메모리, 보조기억장치를 초기화한다.
    - 운영체제를 로딩하기 전 시스템이 정상적으로 동작 가능한지 확인한다.
3. **MBR 확인**
    - MBR은 `Master Boot Record`의 약자이다.
    - 운영체제의 위치 정보를 확인한다.
    - 어떤 저장 장치에서 운영체제를 불러올지 결정하는 데 사용된다.
4. **Boot Code 실행**
    - 부트 코드가 실행된다.
    - VBR을 호출한다.
    - 운영체제 로딩을 준비한다.
5. **운영체제 이미지 적재**
    - 운영체제 이미지를 메모리로 적재한다.
    - 파일 시스템을 표시하고 초기화한다.
6. **커널 이미지 실행**
    - 커널 이미지를 실행한다.
    - 시스템 초기화와 장치 점검을 수행한다.
7. **init 프로세스 실행**
    - init 프로세스를 실행한다.
    - 초기화 작업을 수행한다.
8. **Login Prompt 표시**
    - 로그인 프롬프트가 표시된다.
    - 사용자가 인증을 완료하면 시스템 사용이 가능해진다.
## 2. BIOS란?
BIOS는 `Basic Input Output System`의 약자이다. 컴퓨터가 부팅될 때 CPU가 0번지 데이터를 읽어 실행하면서 BIOS와 연결된다.
### 2.1 BIOS의 역할
- 컴퓨터 부팅 초기에 실행되는 펌웨어이다.
- 하드웨어 점검을 수행한다.
- CPU, 메모리, 보조기억장치 등 주요 장치를 초기화한다.
- 입출력 처리를 담당한다.
- 운영체제 로딩 전 단계에서 필수적으로 동작한다.
- 펌웨어 형태로 저장된다.
- 비휘발성 메모리인 NVRAM 또는 ROM에 저장된다.
- 전원이 차단되어도 설정과 정보가 유지된다.
### 2.2 BIOS가 제공하는 기능
- 부팅 시 화면 출력
- BIOS 환경 설정 접근
- 공장 초기화 기능
- 하드웨어 점검 및 초기화
- 운영체제 로딩 준비
## 3. 디지털 포렌식에서의 BIOS
BIOS는 컴퓨터 하드웨어의 최하위 단계에서 구동되는 소프트웨어이다. 운영체제가 부팅되기 전에 무조건 실행되며, 시스템 초기화와 점검을 수행한다.
### 3.1 BIOS가 포렌식에서 중요한 이유
#### 3.1.1 시간 정보 확인
- 시스템 시계를 확인할 수 있다.
- 시간 서버 없이 내부 시간을 활용할 수 있다.
- 시스템의 기준 시간이 조작되었는지 확인하는 데 도움을 줄 수 있다.
#### 3.1.2 저장 매체 정보 확인
- 하드디스크 구성 정보를 확인할 수 있다.
- SSD 구성 정보를 확인할 수 있다.
- 저장 매체의 용량을 확인할 수 있다.
- 저장 장치의 장착 여부를 확인할 수 있다.
#### 3.1.3 부팅 정보 확인
- 부팅 우선순위를 확인할 수 있다.
- 운영체제 선택 정보를 확인할 수 있다.
- 부팅 모드를 확인할 수 있다.
### 3.2 BIOS 정보의 포렌식적 특징
- BIOS 정보는 운영체제보다 원시적인 단계의 정보이다.
- 운영체제보다 낮은 계층에서 수집 가능한 데이터이다.
- 정확하고 신뢰 가능한 데이터를 제공할 수 있다.
## 4. BIOS vs. UEFI
BIOS와 UEFI는 모두 운영체제를 구동하기 위한 필수 펌웨어이다. 다만 UEFI는 BIOS의 한계를 개선한 차세대 펌웨어이다.
### 4.1 BIOS
BIOS는 오래된 방식의 펌웨어이며, 주로 텍스트 기반 인터페이스를 사용한다.
#### 4.1.1 BIOS의 특징
- 텍스트 기반 인터페이스를 제공한다.
- 오래된 소프트웨어 방식이다.
- 하드웨어 초기화를 수행한다.
- 부팅 전 건강검진 역할을 수행한다.
- 부트로더를 실행한 뒤 운영체제를 로딩한다.
- 운영체제 로딩까지의 단계가 길고 복잡하다.
#### 4.1.2 BIOS의 제한점
- 대용량 하드디스크를 지원하지 못하는 한계가 있다.
- 부팅 속도가 느리다.
- UI가 불편하다.
### 4.2 UEFI
UEFI는 BIOS의 차세대 펌웨어이다. BIOS보다 현대적인 구조를 가지며, GUI 기반 인터페이스와 개선된 부팅 성능을 제공한다.
#### 4.2.1 UEFI의 특징
- BIOS의 차세대 펌웨어이다.
- GUI 기반 인터페이스를 제공한다.
- 대용량 하드디스크를 지원한다.
- 부팅 속도를 개선한다.
- 부팅 과정을 간소화한다.
- BIOS 단계를 통합한다.
#### 4.2.2 UEFI의 보안 기능
- 무결성 검증 기능을 제공한다.
- 검증된 부팅 이미지만 실행할 수 있다.
- 부팅 단계에서 보안을 강화한다.
### 4.3 BIOS와 UEFI의 공통점
- CPU 전원 인가 시 0번지 데이터를 읽고 실행한다.
- 운영체제를 구동하기 위한 필수 펌웨어이다.

# 2차시: 레지스트리 분석
## 1. 윈도우 부팅 단계
윈도우 부팅 단계는 Boot Manager에서 시작해 Winload, 커널 초기화, 세션 관리자, 로그인 및 Explorer 초기화로 이어진다. 각 단계는 포렌식 관점에서 확인 가능한 흔적을 남길 수 있다.
### 1.1 Boot Manager
- BIOS 또는 UEFI가 부팅 장치를 확인한다.
- 이후 제어권을 Boot Manager에게 전달한다.
- Boot Manager는 BCD를 로딩한다.
- BCD는 `Boot Configuration Data`의 약자이다.
- BCD 로딩을 통해 운영체제와 부팅 모드를 결정한다.
### 1.2 Winload.exe / Winload.efi
- 커널을 메모리에 로딩한다.
- HAL을 메모리에 로딩한다.
- 핵심 드라이버를 메모리에 로딩한다.
- 안전모드와 복구 모드 등의 설정이 적용될 수 있다.
### 1.3 Kernel Init
- 커널을 초기화한다.
- 서브시스템을 초기화한다.
- 운영체제의 제어권이 커널로 전달된다.
### 1.4 Session Manager: smss.exe
- 사용자 세션을 생성한다.
- 레지스트리를 초기화한다.
- 시작 프로그램을 로딩한다.
### 1.5 Windows Logon / Explorer
- 사용자가 로그인을 완료한다.
- 사용자 환경이 구성된다.
- 데스크탑 인터페이스가 초기화된다.
- 이벤트 로그와 레지스트리를 통해 관련 흔적을 확인할 수 있다.
### 1.6 포렌식 포인트
- BCD 흔적을 확인할 수 있다.
- Winload 흔적을 확인할 수 있다.
- 이벤트 로그를 확인할 수 있다.
- 레지스트리를 확인할 수 있다.
- 세션 정보를 확인할 수 있다.
## 2. 레지스트리란?
레지스트리는 윈도우 전용 데이터베이스이다. 윈도우 운영체제의 부팅, 하드웨어, 시스템 설정, 사용자 환경 정보 등을 저장한다.
### 2.1 레지스트리의 기본 특징
- 윈도우 전용 데이터베이스이다.
- 부팅 관련 정보를 저장한다.
- 하드웨어 정보를 저장한다.
- 시스템 설정 정보를 저장한다.
- 사용자 환경 정보를 저장한다.
- Hive 파일 단위로 저장된다.
- 계층적 구조를 가진다.
- 주요 하이브로 HKLM, HKCU, HKCR 등이 있다.
### 2.2 Hive 파일 단위 저장
레지스트리는 Hive 파일 단위로 저장된다. 대표적인 Hive 파일은 다음과 같다.
- `SYSTEM`
- `SOFTWARE`
- `SAM`
- `SECURITY`
- `NTUSER.DAT`
### 2.3 레지스트리 계층 구조
레지스트리는 다음과 같은 계층적 구조를 가진다.
```text
Hive → Key → Value
```
- **Hive**: 레지스트리의 큰 저장 단위이다.
- **Key**: 하이브 내부의 폴더 또는 경로와 같은 구조이다.
- **Value**: 실제 설정값 또는 데이터가 저장되는 단위이다.
## 3. 주요 Hive / 키
### 3.1 HKEY_LOCAL_MACHINE: HKLM
- 시스템 정보를 저장한다.
- 하드웨어 정보를 저장한다.
- 드라이버 정보를 저장한다.
- 서비스 정보를 저장한다.
### 3.2 HKEY_CURRENT_USER: HKCU
- 현재 로그인한 사용자의 환경 설정을 저장한다.
- 사용자별 설정 확인에 사용된다.
### 3.3 HKEY_USERS: HKU
- 사용자 계정별 프로파일 정보를 저장한다.
- 여러 사용자 계정의 흔적을 확인할 때 사용된다.
### 3.4 HKEY_CLASSES_ROOT: HKCR
- 파일 확장자 정보를 저장한다.
- 프로그램 연결 정보를 저장한다.
- 특정 파일 형식이 어떤 프로그램과 연결되어 있는지 확인할 수 있다.
### 3.5 HKEY_CURRENT_CONFIG: HKCC
- 현재 하드웨어 프로필 정보를 저장한다.
- 현재 시스템 구성과 관련된 정보를 확인할 수 있다.
## 4. 포렌식 관점에서의 레지스트리
레지스트리는 사용자의 행위와 시스템 설정을 추적할 수 있는 핵심 아티팩트이다.
### 4.1 추적 가능한 정보
- 시작 프로그램 정보
- 연결된 장치 정보
- 사용자 계정 설정
- 시스템 설정 정보
- 하드웨어 및 드라이버 정보
- 서비스 정보
### 4.2 포렌식 활용 의미
- 사용자가 어떤 프로그램을 실행했는지 확인할 수 있다.
- 어떤 장치가 연결되었는지 확인할 수 있다.
- 어떤 사용자 계정이 사용되었는지 확인할 수 있다.
- 악성코드의 자동 실행 설정 여부를 확인할 수 있다.
- 시스템 구성 변경 흔적을 확인할 수 있다.

# 3차시: 웹 브라우저 분석
## 1. 레지스트리 기반 사용자 행위 흔적
3차시 자료에서는 제목상 웹 브라우저 분석을 다루지만, 앞부분에서는 레지스트리를 통해 확인 가능한 사용자 행위 흔적을 정리한다.
## 2. 프로그램 실행 흔적
### 2.1 UserAssist
- 경로는 다음과 같다.
```text
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\UserAssist
```
- ROT13으로 인코딩된 프로그램 실행 기록을 저장한다.
- 프로그램 목록을 확인할 수 있다.
- 프로그램 실행 횟수를 확인할 수 있다.
- 마지막 실행 시간을 확인할 수 있다.
### 2.2 자동 실행 프로그램 설정값
자동 실행 프로그램은 사용자가 로그인하거나 시스템이 시작될 때 자동으로 실행되는 프로그램 설정을 의미한다.
#### 2.2.1 사용자별 자동 실행 경로
```text
HKCU\…\Run
```
- 현재 사용자 기준 자동 실행 항목을 확인할 수 있다.
#### 2.2.2 모든 사용자 공통 자동 실행 경로
```text
HKLM\…\Run
```
- 시스템 전체 사용자에게 공통으로 적용되는 자동 실행 항목을 확인할 수 있다.
#### 2.2.3 포렌식 의미
- 악성코드가 지속 실행을 위해 자주 악용하는 위치이다.
- 자동 실행 설정이 있다고 해서 실제 실행되었다고 단정하면 안 된다.
- 실제 실행 여부는 UserAssist와 Prefetch를 함께 확인해 교차 검증해야 한다.
## 3. 문서/파일 접근 기록
### 3.1 RecentDocs
- 경로는 다음과 같다.
```text
HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\RecentDocs
```
- 최근 열었던 문서와 파일 목록을 저장한다.
- LastWriteTime을 통해 근사 시간을 확인할 수 있다.
### 3.2 OpenSavePidlMRU / LastVisitedPidlMRU
- 문서를 열거나 저장할 때 사용한 실제 경로 정보를 기록한다.
- 어떤 문서를 어떤 경로에서 접근했는지 파악할 수 있다.
- 단순히 파일명만 보는 것이 아니라 접근 위치를 추적하는 데 유용하다.
## 4. 폴더 접근 기록
### 4.1 BagMRU
- 경로는 다음과 같다.
```text
HKCU\Software\Microsoft\Windows\Shell\BagMRU
```
- 폴더 열람 이력을 저장한다.
- 폴더 뷰 설정을 저장한다.
- 삭제된 폴더의 흔적도 유지될 수 있다.
- USB 내부 폴더의 기록도 유지될 수 있다.
- 사용자의 폴더 접근 행위를 보여주는 블랙박스 역할을 한다.
### 4.2 TypedPaths
- 경로는 다음과 같다.
```text
HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\TypedPaths
```
- 탐색기 주소창에 사용자가 직접 입력한 경로를 기록한다.
- 사용자가 특정 경로에 의도적으로 접근했는지 판단하는 데 활용할 수 있다.
## 5. USB 연결 기록
## 5.1 USBSTOR
- 경로는 다음과 같다.
```text
HKLM\SYSTEM\CurrentControlSet\Enum\USBSTOR
```
- 연결된 USB 장치명을 기록한다.
- 제조사 정보를 기록한다.
- 시리얼 번호를 기록한다.
- 과거 연결된 외부 장치를 확인할 수 있다.
## 5.2 MountedDevices
- 경로는 다음과 같다.
```text
HKLM\SYSTEM\MountedDevices
```
- 각 장치가 어떤 드라이브 문자로 연결되었는지 저장한다.
- USBSTOR 정보와 결합하면 특정 USB가 어떤 경로에 흔적을 남겼는지 추적할 수 있다.
### 5.3 USB 연결 기록의 포렌식 의미
- 어떤 USB가 연결되었는지 확인할 수 있다.
- 해당 USB의 제조사와 시리얼 번호를 확인할 수 있다.
- USB가 어떤 드라이브 문자로 마운트되었는지 확인할 수 있다.
- USB 내부 폴더 접근 기록과 결합해 파일 유출이나 외부 저장장치 사용 정황을 분석할 수 있다.
## 6. 웹 브라우저 아티팩트
웹 브라우저 아티팩트는 사용자의 웹 사용 행위를 복원하는 데 중요한 자료이다. 방문 기록, 캐시, 다운로드 기록, 쿠키, 로그인 및 자동완성 정보가 주요 분석 대상이다.
## 7. 방문 기록
- Chrome, Edge, Firebase 등은 방문 기록을 SQLite DB 형태로 저장한다.
- 방문한 사이트를 확인할 수 있다.
- URL을 확인할 수 있다.
- 검색어 기록을 확인할 수 있다.
- 마이크로초 단위의 시간 기록을 제공한다.
- 정확한 타임라인 복원이 가능하다.
## 8. 캐시
- HTML 리소스를 저장한다.
- 이미지 리소스를 저장한다.
- JavaScript 파일을 저장한다.
- 폰트 등 웹 리소스를 저장한다.
- 사용자가 히스토리를 지워도 캐시 시간 기록을 통해 방문 사실을 확인할 수 있다.
- 기록 삭제 여부를 판단하는 데 활용할 수 있다.
- 증거 인멸 여부를 판단하는 데 활용할 수 있다.
## 9. 다운로드 기록
- 웹 브라우저 DB에 과거 다운로드 파일 목록이 저장된다.
- 파일명을 확인할 수 있다.
- 파일 위치를 확인할 수 있다.
- 다운로드 시간을 확인할 수 있다.
- 사용자가 어떤 파일을 내려받았는지 분석할 수 있다.
## 10. 쿠키
- 세션 유지 정보를 저장한다.
- 자동 로그인 정보와 관련될 수 있다.
- 사용자의 브라우징 패턴을 분석할 수 있다.
- 사용자의 행위를 분석할 수 있다.
- 세션 타임값을 통해 접속 시점을 추정할 수 있다.
## 11. 로그인 / 자동완성 정보
- 웹사이트 로그인 정보를 저장한다.
- 자동완성 데이터를 저장한다.
- 자동완성 데이터에는 이름, 주소, 이메일 등이 포함될 수 있다.
- 계정 사용 흔적을 파악할 수 있다.
- 접속 행위를 분석할 수 있다.
## 12. 3차시 분석 관점 정리
3차시의 핵심은 레지스트리와 웹 브라우저 DB를 통해 사용자의 행위를 복원하는 것이다.
### 12.1 레지스트리 기반 행위 분석
- 프로그램 실행 여부 확인
- 자동 실행 설정 확인
- 문서 및 파일 접근 기록 확인
- 폴더 접근 기록 확인
- 사용자가 직접 입력한 경로 확인
- USB 연결 및 마운트 흔적 확인
### 12.2 웹 브라우저 기반 행위 분석
- 방문한 사이트 확인
- 검색어 확인
- 방문 시각 기반 타임라인 복원
- 캐시를 통한 삭제된 방문 흔적 확인
- 다운로드 파일 확인
- 쿠키를 통한 세션 및 접속 시점 추정
- 로그인 및 자동완성 정보를 통한 계정 사용 흔적 분석
### 12.3 교차 검증의 중요성
단일 아티팩트만으로 행위를 단정하면 위험하다. 예를 들어 자동 실행 레지스트리 값은 프로그램이 자동 실행되도록 설정되어 있음을 보여주지만, 실제 실행 여부를 직접 증명하지는 않는다. 따라서 UserAssist, Prefetch, 브라우저 DB, 캐시, 다운로드 기록, USB 연결 기록 등을 함께 비교해야 한다.
