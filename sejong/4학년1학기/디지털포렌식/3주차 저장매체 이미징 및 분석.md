# 1. 저장매체 이미징의 이해
## 1.1 증거 수집 기법의 분류
### 1.1.1 디스크 이미징
- 비휘발성 저장매체(Non-Volatile Storage) 전체를 원본 그대로 복제하는 기법이다.
### 1.1.2 라이브 포렌식:
- 전원이 꺼지면 사라지는 휘발성 데이터(Volatile Storage)를 실시간으로 수집하고 분석하는 기법이다.
### 1.1.3 디스크 이미징 vs 라이브 포렌식 비교
- **분석 대상**
    - 디스크 이미징: 비휘발성 저장 매체 (디스크)
    - 라이브 포렌식: 휘발성 데이터 (메모리, 프로세스, 네트워크 상태 등)
- **데이터 안정성**
    - 디스크 이미징: 높음
    - 라이브 포렌식: 낮음
- **긴급성 및 실시간성**
    - 디스크 이미징: 낮음
    - 라이브 포렌식: 높음
- **데이터 손상 위험성**
    - 디스크 이미징: 낮음
    - 라이브 포렌식: 높음
## 1.2 디스크 이미징의 개념과 중요성
### 1.2.1 디스크 이미징이란?
- 비휘발성 저장매체에 저장된 모든 데이터를 비트 단위로 복제하여 정확한 사본을 생성하는 과정이다.
- **Bit Stream Image**라고도 부른다.
### 1.2.2 디스크 이미징의 특징
- 원본 데이터는 물론, 메타데이터와 삭제되거나 비어 있는 공간(빈 공간 데이터)까지 모두 포함하여 복제한다.
- 원본의 손상 없이 생성된 사본을 통해 반복적인 분석이 가능하다.
- 법적 증거로서의 신뢰성과 진정성을 보장한다.
### 1.2.3 디스크 이미징의 중요성
- 훼손 우려 없이 반복적인 분석이 가능하다.
- 데이터의 신뢰성과 진정성을 보장한다.
- 디지털 증거 확보 과정에서 반드시 거쳐야 하는 필수 절차이다.
## 1.3 디스크 이미징 도구의 종류와 특징
### 1.3.1 도구의 종류
- **소프트웨어 방식**
    - **FTK Imager**: GUI(그래픽 사용자 인터페이스) 기반 도구이다.
    - **EnCase**: 데이터 수집과 분석 기능을 모두 제공한다.
    - **Linux 명령어**: `dd`, `dcfldd` 등 CLI(명령줄 인터페이스) 기반 도구이다.
- **하드웨어 방식**
    - 디스크 이미징 전용 장비를 사용한다.
    - 안정적이고 빠른 복제가 가능하며 이동성이 용이하다.
    - 원본 디스크에서 이미징 디스크로 직접 복제하거나 네트워크 전송이 가능하다.
### 1.3.2 도구의 특징
- 소프트웨어 도구는 사용 환경에 따라 GUI 또는 CLI 형태로 선택하여 사용할 수 있다.
- 하드웨어 도구는 단독 복제 작업 및 전문적인 포렌식 장비로 활용된다.
- **기본 원리**: 모든 도구는 본질적으로 '비트 단위 복제'를 수행하며, 내부적으로는 `dd` 명령어 기반으로 동작할 수 있다.
# 2. 실습환경 구성 및 디스크 이미징 실습
## 2.1 가상머신 기반 실습 환경 구축
### 2.1.1 실습 환경 구성 목적
- 실제 운영 환경이 아닌 안전한 격리된 실습 환경을 마련하기 위함이다.
- 다양한 침해 및 포렌식 상황을 재현하여 실습할 수 있다.
### 2.1.2 필요 소프트웨어
- **VirtualBox**: 가상머신을 생성하기 위한 소프트웨어이다. Windows, Linux, Mac OS를 모두 지원하며, 설치 후 가상머신을 생성할 수 있다.
## 2.2 가상머신 다운로드 및 구동
### 2.2.1 실습용 운영체제
- **Kali Linux**: Debian 기반의 리눅스 배포판이다.
- 모의해킹 및 다양한 보안 분석 도구가 기본적으로 포함되어 있어 Offensive 실습용으로 특화되어 있다.
### 2.2.2 가상 디스크 구성
- **수집 대상 디스크 (1GB)**: 실제 디스크 이미징 대상이 되는 증거 디스크이다.
- **수사관 디스크 (2GB)**: 추출한 이미징 파일 및 메타데이터를 저장하기 위한 넉넉한 용량의 디스크이다.
## 2.3 수사대상 디스크 식별 및 생성
### 2.3.1 가상머신 부팅
- 운영체제: Kali Linux
- 디폴트 계정 정보: ID = `KALI` / PW = `KALI`
- 루트 계정으로 전환: `su root` 명령어를 사용한다.
### 2.3.2 디스크 확인
- 명령어: `fdisk -lu`
- **확인 결과**:
    - `sdb`: 1GB 크기의 분석 대상 디스크이다.
    - `sdc`: 2GB 크기의 이미징 저장용 디스크이다.
### 2.3.3 수집 대상 디스크 데이터 생성
- 명령어: `dd if=/dev/urandom of=/dev/sdb count=2`
- 1GB 디스크 공간 전체가 무작위(Random) 데이터로 가득 채워지며, 이를 통해 분석용 가상 디스크 준비가 완료된다.
## 2.4 수집 디스크, 파티션, 파일 시스템 생성
### 2.4.1 디스크 초기화
- 대상: 2GB 수사관 디스크 (`sdc`)
- 명령어: `dd if=/dev/zero of=/dev/sdc bs=16K`
- 디스크 전체가 0으로 덮어씌워져 초기화되며, 이미징 저장 준비가 완료된다.
### 2.4.2 파티션 생성
- 명령어: `fdisk`
- 물리적 저장매체를 운영체제가 인식할 수 있는 논리적 단위로 분할한다.
### 2.4.3 파일 시스템 생성
- 명령어: `mke2fs`
- 생성 형식: `ext2`
- 분할된 파티션 디스크를 운영체제에서 인식하고 사용할 수 있는 파일 시스템 형태로 포맷한다.
### 2.4.4 수집 디스크 파일시스템 생성 확인
- 명령어: `fdisk -lu`
- 파티션 분할 및 ext2 파일 시스템이 정상적으로 생성되었는지 확인한다.
## 2.5 수사대상 디스크 마운트 및 해시값 추출
### 2.5.1 디스크 마운트
- 포맷이 완료된 디스크를 운영체제 디렉토리 트리에 연결하여 접근 가능하게 만든다.
- 명령어:
    - `mkdir /mnt/target` (마운트할 디렉토리 생성)
    - `mount /dev/sdb /mnt/target` (생성한 디렉토리에 디스크 연결)
### 2.5.2 원본 디스크 해시값 추출
- 명령어: `sha256sum /dev/sdb > /mnt/target/insuspect1.sha`
- 원본 디스크의 해시값을 추출하고 저장하여 원본의 무결성을 확인한다.
### 2.5.3 디스크 이미징
- 명령어: `dd if=/dev/sdb of=/mnt/target/inspect1.img bs=16K`
- 수집 대상 디스크(`sdb`)를 비트 단위로 그대로 읽어 수사관 디스크 경로에 파일 형태로 저장한다.
### 2.5.4 이미징 파일 해시값 추출
- 명령어: `sha256sum /mnt/target/inspect1.img > /mnt/target/inspect1.img.sha`
- 이미징 파일에 대한 해시값을 추출하여 원본과 대조할 무결성 확인용 데이터를 확보한다.
### 2.5.5 무결성 검증
- 명령어: `cat` (저장된 해시값 텍스트 파일을 출력하여 비교)
- 원본 디스크와 이미징 파일의 해시값이 일치하는지 대조하여 이미징 작업이 정상 수행되었음을 검증한다.
# 3. FTK Imager를 이용한 이미징 파일 분석
## 3.1 FTK Imager를 활용한 디스크 이미징 절차
### 3.1.1 FTK Imager란?
- 스토리지 디바이스에 대한 이미지(사본) 생성이 가능한 도구이다.
- 수집된 디스크 이미지를 대상으로 내부 분석 기능을 제공한다.
- 현재 실행 중인 시스템의 메모리를 그대로 추출하여 라이브 포렌식을 수행할 수 있는 강력한 소프트웨어이다.
### 3.1.2 가상의 수집대상(USB 스토리지) 플러그 인
- 분석할 USB 스토리지를 준비하여 컴퓨터 시스템에 삽입한다.
- 해당 디스크를 타겟으로 지정하여 이미징을 수행한다.
### 3.1.3 FTK Imager 실행 및 디스크 이미징 수행
1. `Create Disk Image` 메뉴를 선택한다.
2. **소스(Source)**: `Physical Drive`를 선택한 후, 삽입한 USB 드라이브를 지정한다.
3. **이미지 타입**: `Raw` 형식을 선택한다 (디스크의 비트 배열을 그대로 저장).
4. Case Number, Evidence Number, Description 등 사건 및 증거 관련 정보를 입력할 수 있다.
5. 저장 위치를 설정한다.
6. **이미징 진행**: `Start` 버튼을 클릭하면 디스크 이미지 파일과 관련 텍스트 로그 파일이 자동으로 생성된다.
### 3.1.4 FTK Imager 이미징 파일 검증
- 생성된 결과 로그 파일에 해시값, 조사자 정보, 사건 번호 등이 자동으로 기록된다.
- 이미징 작업이 완료된 후 정상적으로 생성되었다는 메시지를 확인한다.
## 3.2 dd 명령어를 이용한 디스크 이미징 절차
- Windows 환경용 dd 유틸리티 다운로드: `http://www.chrysocome.net/downloads/dd-0.5.zip`
- 이전 2차시에서 실습했던 것과 동일한 원리로, 물리적 디스크를 대상으로 CLI 기반의 이미징을 수행하고 결과를 확인할 수 있다.
## 3.3 FTK Imager를 이용한 이미징 파일 분석
### 3.3.1 파일 분석 진행
- 수집된 디스크 이미징 파일(예: `alienimage.dd`)을 FTK Imager로 불러와 분석한다.
- 파일 시스템의 구조와 내부 데이터, 폴더 트리를 직접 탐색한다.
### 3.3.2 FTK Imager 주요 기능
- **삭제 파일 확인**: 삭제된 파일은 `X` 아이콘으로 표시되며, 해당 파일의 복구 가능 여부를 직관적으로 확인할 수 있다.
- **MAC Time 확인**: 각 파일의 수정(Modified), 접근(Accessed), 생성(Created) 날짜 및 시간을 상세히 확인할 수 있다.
- **미리보기(Preview)**: 파일 내부의 데이터를 렌더링하여 미리보기 형태로 제공한다.
- **Hex 뷰**: 이미지, 문서 등의 파일을 Hex(16진수) 형태로 열어 내부의 원시 데이터를 정밀 분석할 수 있다.
## 3.4 Autopsy를 이용한 이미징 파일 분석
### 3.4.1 Autopsy란?
- 널리 사용되는 오픈소스 기반의 디지털 포렌식 도구이다.
- GUI를 기반으로 작동하며, 내부적으로는 강력한 Sleuth Kit 명령어 툴킷을 기반으로 동작한다.
- 하드디스크, USB, 스마트폰 이미지 등 매우 다양한 저장장치의 분석이 가능하다.
### 3.4.2 Autopsy 주요 기능
- 파일 복구 및 내부 디렉토리 탐색 기능
- 타임라인(Timeline) 분석을 통한 사건의 시간적 재구성
- 웹 활동, 브라우저 쿠키, 이메일, 채팅 기록 등에 대한 특화된 자동 분석 지원
- 이미지 파일 및 비디오 파일의 썸네일(Thumbnail) 모아보기 기능 제공
