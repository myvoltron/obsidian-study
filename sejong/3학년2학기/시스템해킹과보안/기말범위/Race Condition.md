# 1. Race Condition의 기본 개념
- 프로세스 또는 스레드 간에 자원을 사용하기 위한 경쟁
- "관리자 권한"으로 생성되고 사용되는 **임시 파일**에 대한 치환(Substitution, 끼워넣기)을 이용해 공격.
## 1.2. 주요 Race Condition 취약점: TOCTOU
- **정의:** 소프트웨어에서 **리소스의 상태를 확인하는 시간**(Time-of-Check)과 **실제 리소스를 사용하는 시간**(Time-of-Use) 사이에 공격자가 해당 리소스를 변경하여 상태 확인 결과를 우회하는 방식입니다.
- **결과:** 리소스가 예기치 않은 상태로 변경되어 소프트웨어가 잘못된 작업을 수행할 수 있습니다.
# 2. 하드 링크와 심볼릭 링크
## 2.1 하드 링크
- 두 파일이 **동일한 I-node를 공유**하며 데이터를 동기화함. 링크된 파일을 수정하면 원본 파일도 수정됨.
- 파일 삭제 시 **링크 카운트**가 1 감소함. (링크 카운트가 0이 되면 삭제됨)
- 링크할 파일은 같은 파티션에 있어야 함.
## 2.2 심볼릭 링크
- 원본 파일을 참조하는 **링크 정보**만 가짐. 
- 데이터를 가진 파일은 원본 파일 하나뿐입니다. 
- `ln` 명령어에 `-s` 옵션을 사용하여 생성합니다.
- 원본 파일을 삭제하더라도 심볼릭 링크는 원본 파일의 경로를 계속해서 가리킴.
# 3. 심볼릭 링크와 Race Condition 공격
## 3.1 Requirements
- 소유자가 root, SetUID 비트, 임시 파일을 생성하는 프로그램
## 3.2 정상적인 프로그램 실행 절차
1. (정상 프로세스) 임시 파일 존재 여부 확인.
2. (정상 프로세스) 파일이 존재하면 삭제하고 다시 생성.
3. (정상 프로세스) 임시 파일 액세스.
## 3.3 Race Condition 공격 과정 (공격 성공 시):
공격 코드는 `unlink()`를 사용하여 기존 심볼릭 링크를 해제하고, `symlink()`를 사용하여 공격 대상 파일(예: `/etc/passwd`)로 다시 링크하는 작업을 반복합니다.
1.  (정상 프로세스) 임시 파일이 존재하는지 확인합니다.
2. (정상 프로세스) 파일이 존재하므로 삭제하고 재생성하려고 시도합니다.
3. **이 두 단계 사이에 공격자가 개입합니다.**
4. (공격 프로세스) 임시 파일이 존재하는지 확인하고 심볼릭 링크인지 확인합니다.
5. (공격 프로세스) 심볼릭 링크가 아니면 삭제합니다.
6. (공격 프로세스) 임시 파일을 **목표 파일로 연결되는 심볼릭 링크로 생성**합니다.
7. (정상 프로세스) **관리자 권한으로 실행되던 프로세스**가 이제 심볼릭 링크를 따라 **목표 파일**을 액세스하여 데이터를 씁니다.
# 4. Race Condition에 대한 대응 방안 (Countermeasures)
## 4.1 안전한 파일 열기
1. `if(lstat(filename, &st) != 0)`: 파일의 링크 상태를 확인
2. `if(!S_ISREG(st.st_mode))`: 열고자 하는 파일 종류를 확인한다.
3. `if(st.st_uid != 0)`: 열고자 하는 파일의 사용자 계정 검사.
4. `if((st.st_ino != st2.st_ino) || (st.st_dev != st2.st_dev))`: 최초 파일 정보를 저장하고 있는 st, 파일은 연 후의 st2에 저장된 I-node 값, 장치 값을 변경했는지 확인.
## 4.2 커널 보안 강화
- 리눅스 커널은 안전하지 않은 파일 시스템 액세스 취약점(tmp-races, TOCTOU)의 위협을 완화하기 위해 보안 기술을 적용했습니다.
    - 예: Ubuntu 16.04에서 `fs.protected_symlinks=1`을 설정하여 심볼릭 링크 보호를 활성화할 수 있습니다.
