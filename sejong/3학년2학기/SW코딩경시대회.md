# 2. 문제 제목: 다중 암호 해독기 (Multi-Delimiter ASCII Decoder)
- 비슷한 문제를 찾을 수 없어서 chatGPT로 복원. 
## 문제 설명
어떤 문자열 ciphertext가 주어진다.
이 문자열에는 특수한 기호(* , #, &)가 섞여 있으며, 이들 기호 사이에는 숫자(ASCII 코드) 가 숨겨져 있다.
당신의 임무는 이 문자열을 분석하여 다음 규칙에 따라 해독한 문자를 answer 문자열로 만들어 출력하는 것이다.
## 규칙
1. * ... *
    → 사이의 숫자를 ASCII 코드로 변환하고, 변환된 문자를 **그대로** 결과에 추가한다.
    예: *65* → 'A'
2. # ... #
    → 사이의 숫자를 ASCII 코드로 변환하고, 변환된 문자가 **대문자(****A****~****Z****)일 경우에만** 결과에 추가한다.
    예: #65# → 'A' 추가, #97# → 'a' 무시
3. &# ... #&
    → 이 구간은 **반전 구간**이다.
    - 내부 숫자는 ASCII 코드로 변환하지만, 변환된 문자가 **소문자(****a****~****z****)일 경우에만** 결과에 추가한다.
    - 즉, #...#의 반대 조건이다.
        예: &#97#& → 'a' 추가, &#65#& → 'A' 무시
4. #& ... &#
    → 이 구간은 **무시 구간**이다.
    - 이 안에 있는 모든 문자는 변환하지 않고 무시한다.
        예: *65*#&*66*#90#&#*67* → *65*#90#만 처리됨
## 입력 형식
- 첫 줄에 암호문 문자열 ciphertext가 주어진다.
- 문자열 길이: 1 이상 2000 이하
- ciphertext는 알파벳, 숫자, 공백, 특수문자(*, #, &)로 이루어져 있다.
- 입력에는 잘못된 구간(짝이 안 맞는 구분자)은 주어지지 않는다.
## 출력 형식
- 위 규칙에 따라 해독된 문자열을 한 줄에 출력한다.
- 변환 결과가 없다면 빈 문자열을 출력한다.
## 예제 1
**입력**
```
abc*65*#97#*66*#&*67*#90#&#*68*&#97#&xyz
```
**출력**
```
ABa
```
**설명**
- *65* → 'A'
- #97# → 'a' (소문자 → 무시)
- *66* → 'B'
- #&*67*#90#&# 구간은 무시됨
- &#97#& → 'a' 추가
## 예제 2
**입력**
```
*72*#101#*108*&#108#&*111*
```
**출력**
```
Hllo
```
**설명**
- *72* → 'H'
- #101# → 'e' (소문자 → 무시)
- *108* → 'l'
- &#108#& → 'l' (소문자 → 추가됨)
- *111* → 'o'
