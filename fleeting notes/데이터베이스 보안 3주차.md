# 1장. 관계형 데이터베이스(RDB) 개요
## 1.1 RDB 정의 및 예시
- 관계형 모델은 1970년대 IBM의 E. F. Codd가 제안한 모델로, 데이터를 행(row)과 열(column)로 구성된 **테이블 형태**로 표현함.
- 키(Key)와 값(Value)을 표 형태로 구성한 데이터베이스
- 예시: Oracle, SQL Server, DB2, MariaDB, PostgreSQL, SQLite 등 다양한 상용/오픈소스 관계형 DBMS 존재
## 1.2 RDB 특징
### 1.2.1 특징에 따른 장점
- 단순한 표 형태로 데이터 저장 -> 이해 쉬움
- SQL로 데이터 관리 -> 데이터 이용 쉬움, 확장 쉬움
- 다양한 보안 기능 -> 데이터 공유 환경에서 효과적
### 1.2.2 특징에 따른 단점
- 처음 설계한 데이터베이스 구조 변경 또는 확장 -> 변경된 구조 최적화 및 관리에 많은 비용 소요
## 1.3 RDB 모델링
### 1.3.1 릴레이션의 구성
- **릴레이션(Relation)**: 하나의 테이블
- 릴레이션 스키마(Relation Schema)(= 릴레이션 내포(Relation Intension))
    - 릴레이션의 논리적 구조.
    - 릴레이션 이름 + 릴레이션 내의 속성 이름으로 정의 (예: 고객(고객아이디, 이름, 등급)) => 속성의 집합
    - 정적임
- **릴레이션 인스턴스(Relation Instance)(= 릴레이션 외연(Relation Extension))**
    - 특정 시점에 릴레이션에 저장된 데이터 집합 => 튜플의 집합
    - 튜플의 처리가 자주 발생 -> 동적임
### 1.3.2 데이터베이스의 구성
- **데이터베이스 스키마**: 데이터베**이스의 전체 구조, 릴레이션 스키마의 집합**
- **데이터베이스 인스턴스**: 릴레이션 인스턴스의 집합
### 1.3.3 릴레이션 특성
- 튜플의 유용성 - 튜플 중복 X -> 기본키가 제공됨
- 튜플의 무순서 - 순서 무의미
- 속성의 무순서 - //
- 속성의 원자성 - 속성값으로 하나의 값만 사용 가능
# 2장. Key의 종류
## 2.1 키의 개념
- 릴레이션에서 튜플을 구별하기 위한 식별자 역할을 하는 속성 또는 속성들의 집합
## 2.2 키의 특성
- **유일성**: 모든 튜플이 서로 다른 키 값을 가져야 함
- **최소성**: 식별에 꼭 필요한 최소한의 속성만 포함
## 2.3 키의 종류
### 2.3.1 슈퍼키(Super Key)
- 유일성을 만족하는 속성 또는 속성들의 집합
### 2.3.2 후보키(Candidate Key)
- 유일성과 최소성을 만족하는 속성 집합 (슈퍼키 중 최소성 만족하는 것)
### 2.3.3 기본키(Primary Key)
- 후보키 중에서 대표로 선택된 키 (NULL 불허)
- 튜플 중복성 확인용
### 2.3.4 대체키(Alternate Key)
- 기본키로 선택되지 않은 나머지 후보키
### 2.3.5 외래키(Foreign Key)
- 다른 릴레이션의 기본키를 참조하는 속성 또는 속성들의 집합
- 기본키의 속성 이름은 달라도 도메인 일치 필요
- 여러 개 정의 가능
- 외래키도 기본키 가능
- 같은 릴레이션의 기본키를 참조하는 외래키 정의 가능 ex) 추천고객
# 3장. 무결성 제약 조건
## 3.1 무결성 제약 조건 개념
### 3.1.1 무결성
- 데이터가 정확하고 유효하게 유지되는 상태 = 결함 X
### 3.1.2 무결성 제약 조건
- 데이터의 무결성을 보장하고 일관된 상태로 유지하기 위한 규칙
- 데이터 상태 변화(삽입, 삭제, 수정 등) 중에도 무결성 제약 조건은 지켜져야 함
## 3.2 도메인 무결성 제약조건
- 튜플들이 각 속성의 도메인에 지정된 값만을 가져야 함.
- 속성 값과 관련된 무결성으로 타입, NULL 여부, DEFAULT, CHECK 제약 등을 사용해 지정 가능
## 3.3 개체 무결성 제약조건
- 기본키는 반드시 **NULL 불허**, **유일한 값**만 허용
## 3.4 참조 무결성 제약조건
- 외래키는 참조되는 릴레이션의 기본키와 일치하거나 **NULL**이어야 함
- 부모 릴레이션에 존재하지 않는 값은 자식 릴레이션에 삽입/수정 불가
# 4장. 관계 대수
## 4.1 관계 대수 개념 및 종류
### 4.1.1 관계 대수 개념
- 절차 언어(Procedural Language) - 원하는 데이터를 얻기 위해 릴레이션 처리 과정을 순서대로 기술하는 언어
- 릴레이션 처리 연산자들의 집합 - 일반 집합 연산자와 순수 관계 연산자로 분류
- 폐쇄 속성(Closure property) - 피연산자도 릴레이션이고 연산의 결과도 릴레이션
### 4.1.2 관계 대수 종류
- 순수 관계 연**산자**: 셀렉트(σ), 프로젝트(π), 조인(⋈), 디비전(÷)
    - 릴레이션의 구조와 특성 이용
- **일반 집합 연산자**: 합집합(∪), 교집합(∩), 차집합(−), 카티션 곱(×)
    - 릴레이션이 튜플의 집합이라는 개념 이용
    - 두 릴레이션을 대상으로 연산 수행
## 4.2 Select 연산자 (σ)
- 조건을 만족하는 튜플만 추출 → 수평적 부분집합
- 하나의 릴레이션 대상
- 표현: `σ조건식(릴레이션)` 또는 `릴레이션 where 조건식`
    - 연산자 예시: =, ≠, >, <, ∧, ∨ 등
- 교환 가능: `σa(σb(R)) = σb(σa(R))`
## 4.3 Project 연산자 (π)
- 원하는 속성만 선택 → 수직적 부분집합
- 표현: `π속성리스트(릴레이션)` 또는 `릴레이션[속성리스트]`
- 중복 제거 자동 수행
## 4.4 Division 연산자 (÷)
- 릴레이션2의 모든 튜플과 관련 있는 릴레이션1의 튜플 추출
- 표현: `릴레이션1 ÷ 릴레이션2`
## 4.5 Cartesian Product 연산자 (×)
- 두 릴레이션의 튜플을 모두 조합
- 결과 차수: R 차수 + S 차수 / 결과 카디널리티: R x S
- 교환, 결합법칙 만족
## 4.6 Union 연산자 (∪)
- R 또는 S에 속하는 튜플의 합집합
- 교환, 결합법칙 만족
## 4.7 Intersection 연산자 (∩)
- R과 S 양쪽에 존재하는 튜플의 교집합
- 교환, 결합법칙 만족
## 4.8 Difference 연산자 (−)
- R에는 있지만 S에는 없는 튜플 추출
- 교환, 결합법칙 불만족
## 4.9 Join 연산자 (⋈)
- 공통 속성의 값이 같은 튜플끼리 연결하여 새로운 릴레이션 구성 (동등 조인)
- 표현: `R ⋈ S`
